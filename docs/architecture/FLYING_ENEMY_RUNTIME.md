# Flying Enemy Runtime

Этот документ фиксирует реализацию физического летающего врага из issue #18.

## Состав

```text
FlyingEnemyProfile
FlyingEnemySpawnDirector
FlyingEnemyBehavior
BoardingEnemy
BoardingEnemyRegistry
```

Летающий враг остаётся обычным `BoardingEnemy` для здоровья, регистрации, смерти, награды и статистики. Отличающееся движение и выбор цели находятся в отдельном `FlyingEnemyBehavior`, подключённом через framework issue #17.

## Данные

`FlyingEnemyProfile` является типизированным `Resource` и хранит:

- ссылку на базовый `BoardingEnemyArchetype`;
- скорость полёта;
- высоту зависания;
- дистанцию появления;
- интервал появления;
- дистанцию атаки;
- дистанцию и силу воздушного разделения.

Изменяемое состояние экземпляра в ресурсе не хранится.

## Спавн

`FlyingEnemySpawnDirector` имеет собственный таймер и не проверяет наличие якорей. Враг появляется слева или справа за пределами камеры и сразу регистрируется в общем `BoardingEnemyRegistry`.

Летающие враги:

- не учитываются в наземном лимите;
- не выбирают якорь;
- не входят в очередь подъёма;
- не считаются уже поднявшимися на платформу;
- доступны турелям с момента появления.

## Движение и бой

В состоянии `FLYING` behavior выбирает ближайшего живого защитника и движется к точке зависания над ним. При достижении дистанции атаки переходит в `ATTACKING` и использует общий `MeleeAttackComponent`.

Это означает, что подготовка удара, кулдаун, фиксация цели и фактическое изменение здоровья используют существующий проверенный боевой поток.

## Воздушное разделение

Каждый летающий враг проверяет только другие экземпляры с `FlyingEnemyBehavior`. Внутри `separation_distance` добавляется отталкивающая скорость, пропорциональная близости. Наземные и поднимающиеся враги в воздушном разделении не участвуют.

## Пауза

`EnemyBehaviorComponent` вызывает `_tick_behavior()` только при активной мировой симуляции. Поэтому полёт, разделение и `MeleeAttackComponent.tick()` полностью останавливаются при ручной паузе и выборе карточек.

## Смерть и награда

```text
HealthComponent.depleted
→ BoardingEnemy.kill(reason)
→ FlyingEnemyBehavior.stop()
→ BoardingEnemyRegistry.enemy_removed
→ BoardingRewardController
→ RunEconomy / RunStatistics
```

Behavior и spawn director не начисляют монеты напрямую.

## Представление

До подключения финального ассета `BoardingEnemyVisual` рисует для `flyer` круглое тело с двумя диагностическими крыльями. Представление не выбирает цель и не изменяет симуляцию.

## Тестовая граница

`tests/integration/flying_enemy_scenarios.gd` проверяет:

- полёт без установленных якорей;
- отсутствие в ground/climbing/boarded счётчиках;
- полную остановку во время паузы;
- доступность для `TurretTargetSelector`;
- удаление через общий death flow;
- разделение двух летающих врагов.
