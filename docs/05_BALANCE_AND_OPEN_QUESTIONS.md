# GloryProtect — баланс, параметры и отложенные решения

Этот документ отделяет подтверждённые правила от числовых значений, которые должны быть подобраны во время прототипирования.

Главный принцип:

> Не фиксировать баланс в коде до появления данных плейтестов.

---

# 1. Уже подтверждённые константы правил

Следующие значения являются частью концепции, а не временной рекомендацией:

- пять секций щита;
- пять наземных энергетических шаров;
- три защитника в начале забега;
- три сегмента здоровья у базового защитника;
- четыре якоря: два левых и два правых;
- три уровня силы ветра;
- один медицинский пост максимум;
- две карточки улучшений на выбор;
- первый порог улучшения — 5 монет;
- цена растёт на 5 до 100;
- после 100 цена удваивается;
- базовый меч наносит 1 урон;
- базовая турель наносит 1 урон;
- базовый враг имеет 1 здоровье;
- базовый враг наносит 1 сегмент урона;
- указатель шара появляется при 50% прочности секции или ниже;
- указатель исчезает при подъёме выше 50%;
- любая секция при 0% завершает забег;
- отсутствие всех живых защитников завершает забег.

# 2. Параметры, требующие прототипирования

## 2.1. Платформа

| Параметр | Предварительный диапазон | Статус |
|---|---:|---|
| Количество ячеек платформы | 16–20 | выбрать после серого прототипа |
| Масса платформы | не определена | подобрать по ощущению управления |
| Базовое сопротивление движению | не определено | подобрать вместе с инерцией |
| Максимальная горизонтальная скорость | не определена | ограничить только при необходимости |
| Сила рулевого ввода | не определена | подобрать по ветру 1–3 |
| Ширина камеры | не определена | должна скрывать соседние шары |

## 2.2. Ветер

| Параметр | Предварительный диапазон | Статус |
|---|---:|---|
| Сила ветра 1 | не определена | лёгкая компенсация |
| Сила ветра 2 | не определена | заметная компенсация |
| Сила ветра 3 | не определена | опасное состояние |
| Интервал изменения силы | не определён | должен быть читаемым |
| Интервал смены направления | не определён | не должен ощущаться хаотичным |
| Амплитуда мелких колебаний | не определена | не должна ломать контроль |

## 2.3. Энергетический контакт

| Параметр | Предварительный диапазон | Статус |
|---|---:|---|
| Полуширина зоны контакта | не определена | проверить ручное удержание |
| Дополнительная ширина зоны якорей | не определена | немного шире контакта |
| Скорость восстановления секции | не определена | проверить время риска у шара |
| Визуальная задержка луча | 0 или минимальная | не должна менять механику |

## 2.4. Якоря

| Параметр | Предварительный диапазон | Статус |
|---|---:|---|
| Длительность установки | не определена | должна создавать риск |
| Длительность снятия | не определена | настроить отдельно |
| Таймер перегрузки | 2–3 секунды | подтвердить плейтестом |
| Максимальная длина троса | не определена | зависит от геометрии платформы |
| Скорость возврата сорванного якоря | не определена | должна быть хорошо читаема |
| Допуск длины при завершении установки | не определён | исключить пограничный шум |

## 2.5. Щит и стратегические враги

| Параметр | Предварительный диапазон | Статус |
|---|---:|---|
| Критический порог секции | около 25% | предварительное решение |
| Урон одного стратегического врага | не определён | одинаковый для базовых врагов |
| Скорость стратегического врага | не определена | постоянная в рамках базового типа |
| Начальный размер волны | не определён | настроить под стартовую передышку |
| Интервал между волнами | не определён | постепенно сокращается |
| Темп урона массы у щита | не определён | масса должна уменьшаться постепенно |
| Вероятность объединения групп | не определена | случайная |
| Вероятность разделения группы | не определена | случайная в любой момент |

## 2.6. Физические враги

| Параметр | Предварительный диапазон | Статус |
|---|---:|---|
| Скорость бега по земле | не определена | должна позволять видеть маршрут |
| Скорость подъёма по якорю | не определена | основной параметр риска |
| Минимальная дистанция в очереди | не определена | зависит от размера спрайта |
| Начальный интервал спавна | не определён | поток почти непрерывный |
| Начальный общий наземный лимит | не определён | должен быть небольшим |
| Скорость роста лимита | не определена | плавная |
| Длительность прыжка | не определена | турель должна успевать реагировать |
| Дистанция атаки | не определена | единая для базового врага |
| Кулдаун атаки | не определён | медленнее защитника |

## 2.7. Защитники

| Параметр | Предварительный диапазон | Статус |
|---|---:|---|
| Скорость перемещения | не определена | роли не должны меняться мгновенно |
| Кулдаун меча | не определён | немного быстрее базового врага |
| Радиус локальной защиты поста | не определён | зависит от размера платформы |
| Кулдаун замены погибшего | не определён | проверить восстановимость после потерь |
| Время выхода из двери | не определено | визуальный параметр |

## 2.8. Лечение

| Параметр | Предварительный диапазон | Статус |
|---|---:|---|
| Длительность цикла | около 5 секунд | подтвердить плейтестом |
| Лечебный объём | 1 сегмент | подтверждено |
| Радиус начала лечения | не определён | лекарь должен добраться до цели |

## 2.9. Турели

| Параметр | Предварительный диапазон | Статус |
|---|---:|---|
| Радиус центральной турели | почти вся платформа | подобрать геометрически |
| Радиус боковой турели | половина платформы и пространство за краем | подобрать геометрически |
| Кулдаун выстрела | не определён | не должен заменять весь экипаж |
| Скорость снаряда | не определена | либо мгновенное попадание для прототипа |
| Длительность текущего выстрела | не определена | нужна для неделимого действия оператора |

## 2.10. Старт и продолжительность

| Параметр | Предварительный диапазон | Статус |
|---|---:|---|
| Стартовая безопасная пауза | 2–5 секунд | выбрать по первому UX-тесту |
| Целевая длительность раннего забега | не определена | зависит от сложности |
| Целевая медианная длительность полного забега | не определена | определить после контента |

# 4. Метрики для плейтестов

В тестовых сборках рекомендуется собирать:

- длительность забега;
- причину поражения;
- какая секция разрушилась;
- количество погибших защитников;
- число замен защитников;
- время без назначенного рулевого;
- время ручного удержания без якорей;
- время с одним, двумя, тремя и четырьмя якорями;
- количество обрывов из-за ветра;
- количество мгновенных обрывов из-за длины;
- количество врагов, убитых снятием якоря;
- среднее количество врагов на платформе;
- максимальное количество врагов на земле;
- количество частичных и полных зарядок секций;
- число переназначений ролей;
- время простоя турелей;
- время простоя медицинского поста;
- количество заработанных и потраченных монет;
- количество выбранных карточек;
- время, проведённое на экране карточек.

# 5. Основные балансные риски

## 5.1. Якоря слишком выгодны

Симптомы:

- игрок почти всегда использует четыре якоря;
- абордаж не создаёт достаточной угрозы;
- ручное удержание не используется.

Возможные рычаги:

- плотность абордажа;
- скорость подъёма;
- длительность установки и снятия;
- эффективность свободных бойцов;
- радиус и темп турелей.

## 5.2. Якоря слишком опасны

Симптомы:

- игрок избегает якорей;
- зарядка всегда выполняется только вручную;
- роли якорщиков становятся ненужными.

Рычаги:

- ширина зоны контакта;
- сложность ручного удержания;
- начальная плотность врагов;
- время до первого абордажника;
- сила турелей и экипажа.

## 5.3. Рулевой всегда обязателен

Если игрок никогда не снимает рулевого с поста, роль не создаёт решения.

Рычаги:

- ценность дополнительного свободного бойца;
- периоды полной фиксации;
- количество одновременных угроз;
- полезность временного освобождения рулевого.

## 5.4. Лекарь становится обязательным

Если без лекаря невозможно выжить, все забеги будут развиваться одинаково.

Рычаги:

- скорость лечения;
- частота урона;
- кулдаун замены;
- альтернативные защитные улучшения.

## 5.5. Турели вытесняют ближний бой

Рычаги:

- необходимость отдельного оператора;
- ограниченный радиус;
- темп стрельбы;
- давление на самого турельщика;
- стоимость получения дополнительных турелей.

## 5.6. Миникарта слишком непредсказуема

Случайное разделение масс не должно превращать решения в угадывание.

Нужны:

- читаемая анимация разделения;
- достаточное время на реакцию;
- ограничение чрезмерно частых смен целей;
- визуальное отображение размера новых групп.

# 6. Отложенные спецификации

## 6.1. Полный список улучшений

Будет описан отдельно. Требуется определить:

- названия карточек;
- категории;
- редкость;
- веса выпадения;
- повторяемость;
- максимальные уровни;
- взаимоисключения;
- улучшения экипажа;
- улучшения якорей;
- улучшения платформы;
- улучшения турелей;
- улучшения лечения;
- экономические улучшения.

## 6.2. Дополнительные типы врагов

Пока подтверждён только базовый враг.

Позднее можно рассмотреть:

- более живучих врагов;
- быстрых врагов;
- врагов с другой скоростью атаки;
- врагов, специализирующихся на операторах;
- врагов, влияющих на якоря.

Любой новый тип не должен нарушать базовое правило: обычный успешный удар базового врага наносит 1 сегмент.

## 6.3. Мета-прогрессия

Не определено:

- есть ли постоянные разблокировки между забегами;
- открываются ли новые карточки;
- есть ли стартовые комплекты;
- есть ли постоянные улучшения;
- как сохраняются рекорды.

## 6.4. Платформы и режимы

Не определено:

- существует ли только одна стандартная платформа;
- будут ли альтернативные платформы с тем же размером, но другой компоновкой;
- появятся ли дополнительные режимы игры.

## 6.5. Визуальный стиль

Не определены окончательно:

- художественная стилистика;
- масштаб персонажей;
- анимация магического удержания;
- вид стратегической миникарты;
- цветовая палитра секций;
- форма интерфейса.

## 6.6. Звук

Требуется отдельная звуковая спецификация:

- ветер трёх уровней;
- перегрузка якоря;
- обрыв;
- энергетический контакт;
- критическое состояние щита;
- недоступное действие;
- установка и снятие;
- лечение;
- турель;
- появление карточек;
- поражение.

# 7. Решения, которые не следует менять без пересмотра концепции

Следующие изменения затронут саму основу игры и требуют отдельного дизайнерского обсуждения:

- вертикальное управление платформой;
- процедурное создание бесконечного мира;
- свободная установка якорей в любом месте;
- прохождение врагов на платформу без якорей;
- расширение платформы во время забега;
- прямое связывание повреждения щита с физическим спавном;
- автоматическое управление платформой без рулевого;
- мгновенное переназначение защитников;
- лечение нескольких целей одновременно;
- постоянное присутствие якорщика у уже установленного якоря;
- монеты за стратегических врагов;
- продолжение игры при полном уничтожении живого экипажа.
