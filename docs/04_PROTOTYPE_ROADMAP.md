# GloryProtect — план разработки прототипа

Документ задаёт последовательность реализации от минимального физического прототипа до первого вертикального среза. Порядок построен так, чтобы как можно раньше проверить главный игровой конфликт: фиксация платформы якорями против риска абордажа.

---

# 1. Цель прототипа

Прототип должен ответить на пять ключевых вопросов:

1. Интересно ли вручную удерживать платформу против меняющегося ветра?
2. Создаёт ли установка якорей понятный и напряжённый риск?
3. Успевает ли игрок одновременно следить за щитом, платформой и экипажем?
4. Работает ли физическое переназначение защитников как значимое решение?
5. Формирует ли игра повторяемый цикл без необходимости заранее создавать большой объём контента?

---

# 2. Принципы разработки

- Сначала проверяется механика, затем графика.
- Все значения баланса настраиваются через ресурсы.
- Каждый этап должен завершаться играбельной сборкой.
- Новая система не добавляется, пока предыдущая не имеет диагностического UI.
- Для каждого этапа заранее определяются критерии готовности.
- Сложные визуальные эффекты, редкие враги и полный пул улучшений не входят в первый прототип.

# 3. Этап 0 — каркас проекта

## Задачи

- Создать структуру каталогов.
- Создать `GameRoot`.
- Добавить `GameFlowController`.
- Добавить состояния запуска, активной игры, паузы и поражения.
- Создать базовую шину событий или набор типизированных сигналов.
- Создать ресурсы конфигурации баланса.
- Настроить отдельную тестовую сцену механик.

## Результат

Пустая сцена запускается, переходит через стартовую задержку в активное состояние и корректно ставится на полную паузу.

## Критерии готовности

- Нет ошибок при запуске.
- Ручная пауза останавливает тестовый таймер.
- Состояния игры видны в диагностической панели.

# 4. Этап 1 — движение платформы и ветер

## Задачи

- Создать серую платформу фиксированного размера.
- Ограничить движение горизонтальной осью.
- Добавить горизонтальную скорость и управляющую силу.
- Добавить рулевого как условие работы ввода.
- Реализовать ветер направлений влево и вправо.
- Реализовать три уровня силы ветра.
- Добавить небольшие колебания фактической силы.
- Добавить невидимые границы уровня.
- Создать простую камеру, следующую за платформой.
- Добавить диагностические значения скорости, силы и ветра.

## Что тестировать

- Насколько быстро игрок понимает инерцию.
- Насколько трудно удерживать платформу без якорей.
- Не становится ли платформа неуправляемой при ветре 3.
- Достаточно ли заметно изменение ветра.

## Критерии готовности

- Без рулевого стрелки не влияют на платформу.
- С рулевым ввод создаёт силу, а не телепортацию.
- Платформа упирается в границы.
- Ветер разных уровней ощутимо различается.
- Нет накопления скорости из-за ошибок физики при границе.

- Зарядка идёт с постоянной скоростью внутри всей зоны.
- Положение внутри допуска не изменяет скорость.
- Контакт сразу прекращается за границей.

# 6. Этап 3 — точки крепления и четыре якоря

- Добавить четыре точки крепления возле тестового шара.
- Создать расширенную зону доступности.
- Скрывать силуэты вне зоны.
- Показывать зелёные и серые силуэты.
- Добавить отдельную кнопку каждого якоря.
- Реализовать установку с длительностью.
- Реализовать снятие.
- Реализовать отдельные левый и правый посты.
- Добавить очередь установки второго якоря стороны.

## Критерии готовности

- Нельзя установить якорь в произвольном месте.
- Без оператора силуэт серый и команда не ставится.
- Начатая установка завершается после выхода из зоны.
- Очередь второй установки отменяется при переназначении.
- Экстренная команда снимает только обслуживаемые стороны.

# 7. Этап 4 — физика тросов и перегрузка

## Задачи

- Добавить максимальную длину троса.
- Реализовать направленные ограничения.
- Обнулять скорость при полном натяжении.
- Реализовать полную фиксацию при якорях с обеих сторон.
- Исключить маятниковое движение.
- Исключить накопление скрытого рулевого импульса.
- Добавить перегрузку одиночного якоря при ветре 3.
- Добавить красное мигание.
- Добавить таймер 2–3 секунды.
- Добавить отмену перегрузки вторым якорем.
- Добавить отмену перегрузки при ослаблении ветра.
- Добавить возврат сорванного якоря.
- Добавить немедленный обрыв при недостаточной длине после завершения установки.

## Главная проверка этапа

Игрок должен ощущать, что:

- один якорь — быстрый, но ненадёжный вариант;
- два якоря — надёжная фиксация;
- полная фиксация с обеих сторон удобна, но в дальнейшем будет максимально опасна из-за врагов.

## Критерии готовности

- Ветер 1–2 не срывает одиночный якорь.
- Ветер 3 запускает предупреждение, а не мгновенный обрыв.
- Второй якорь полностью сбрасывает перегрузку.
- Платформа не раскачивается.
- После снятия блокировки движение начинается без скрытого импульса.

# 8. Этап 5 — экипаж и переназначение

## Задачи

- Создать трёх защитников.
- Добавить три сегмента здоровья.
- Добавить портреты и выбор нажатием на персонажа.
- Создать контекстное меню ролей.
- Реализовать рулевого, левого и правого якорщика.
- Реализовать физическое перемещение к посту.
- Реализовать ожидание завершения неделимого действия.
- Добавить свободного бойца.
- Добавить смерть.
- Добавить независимые таймеры замены.
- Добавить дверь появления новых защитников.
- Добавить поражение при полном отсутствии живого экипажа.

## Критерии готовности

- Роль активируется только после прибытия.
- Старое неделимое действие не обрывается.
- Несколько защитников могут одновременно ожидать замены.
- Полное уничтожение экипажа немедленно завершает забег.

# 9. Этап 6 — базовый физический враг и ближний бой

## Задачи

- Создать одного базового врага с 1 здоровьем.
- Создать удар защитника с 1 уроном.
- Создать удар врага с 1 уроном.
- Настроить отдельные кулдауны атак.
- Реализовать выбор ближайшей цели.
- Зафиксировать цель начатой атаки.
- Реализовать пустой удар по уже погибшей цели.
- Реализовать локальную зону боя назначенного защитника.
- Реализовать глобальное перемещение свободного бойца.
- Настроить коллизии союзников и противников.

## Критерии готовности

- Защитник обычно выигрывает одиночную дуэль.
- Два врага остаются значимой угрозой.
- Несколько врагов независимо атакуют одну цель.
- Защитники проходят сквозь друг друга.
- Враги создают пробки.

# 10. Этап 7 — наземный спавн и подъём по якорям

## Задачи

- Добавить точки появления за камерой слева и справа.
- Останавливать новый спавн при отсутствии всех якорей.
- Добавить случайный выбор стороны появления 50/50.
- Не компенсировать случайные перекосы.
- Реализовать выбор ближайшего якоря.
- При равной дистанции выбирать случайно.
- Фиксировать выбранную цель.
- Перенаправлять только при недоступности цели.
- Разрешить врагам перебегать под платформой.
- Реализовать следование за платформой.
- Добавить общий плавно растущий лимит.
- Реализовать безвредное состояние без пути.
- Реализовать очереди и подъём по каждому якорю.
- Убивать врагов на тросе при снятии или срыве.

## Критерии готовности

- Без якорей новые враги не появляются.
- Уже существующие враги без пути не наносят урон.
- Каждый якорь является отдельным каналом подъёма.
- Враги не переключаются на новый более близкий якорь без причины.
- Снятие троса не убивает врагов, уже поднявшихся на платформу.

# 11. Этап 8 — прыжок через защитника

## Задачи

- Определять блокировку врага союзником.
- Проверять свободное место за защитником.
- Реализовать прыжковую траекторию.
- Сохранять уязвимость во время прыжка.
- После приземления выбирать ближайшего защитника.
- Не прерывать текущий взмах защитника.

## Критерии готовности

- Один защитник не может навсегда перекрыть проход.
- Прыжок невозможен при занятой посадочной позиции.
- Турель способна убить прыгающего врага.

# 19. Минимальный вертикальный срез

Для первой внешней проверки допустимо временно использовать три шара вместо пяти, но все системы должны быть спроектированы сразу на пять секций.

Вертикальный срез должен содержать:

- одну платформу фиксированного размера;
- три защитника;
- рулевого и два якорных поста;
- четыре якоря;
- ветер 1–3;
- минимум три работающих шара;
- стратегические волны;
- физический абордаж;
- один тип врага;
- свободного бойца;
- медицинский пост;
- одну турель;
- монеты;
- несколько тестовых карточек;
- поражение по щиту и экипажу;
- экран результата.

После подтверждения цикла число шаров в игровой сборке фиксируется на пяти.

# 20. Что не входит в первый прототип

- Полный список улучшений.
- Многоуровневые карточки.
- Несколько типов стратегических врагов.
- Сложные элитные абордажные враги.
- Боссы.
- Мета-прогрессия между забегами.
- Финальная графика и анимации.
- Полная звуковая картина.
- Сюжетные события.
- Локализация.
- Мобильная адаптация интерфейса.

# 21. Рекомендуемый порядок плейтестов

## Плейтест A — движение

Только платформа, ветер и один шар.

Проверяется удовольствие от ручного удержания.

## Плейтест B — якорный выбор

Добавляются четыре якоря без врагов.

Проверяется понятность направленных ограничений и перегрузки.

## Плейтест C — абордаж

Добавляются базовые враги без стратегической угрозы.

Проверяется цена каждого открытого маршрута.

## Плейтест D — многозадачность

Добавляются три секции, миникарта и роли.

Проверяется когнитивная нагрузка.

## Плейтест E — полный цикл

Добавляются пять секций, лечение, турели, экономика и рост сложности.

Проверяются длительность и разнообразие забега.

---

# 22. Признаки успешного прототипа

Прототип считается перспективным, если:

- игрок добровольно использует и ручное управление, и якоря;
- установка второго якоря ощущается значимым решением;
