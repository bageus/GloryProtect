# GloryProtect — статус реализации

## Целевая версия

Godot **4.6.2 stable**, строго типизированный GDScript.

## Текущая версия

**Prototype 1.5 — статистика забега, итоговый экран и критическое предупреждение.**

---

## 1. Статистика забега

Компоненты:

```text
RunStatisticsSnapshot
RunStatistics
SessionRecordsStore
```

### RunStatistics

Единственный владелец:

```text
текущего количества физических убийств
финального снимка завершённого забега
```

Время выживания не дублируется. Оно читается из `RunDifficulty`, который остаётся единственным владельцем активного времени.

Убийство учитывается по сигналу:

```text
BoardingRewardController.reward_granted
```

Поэтому в статистику входят только физические смерти, которые считаются награждаемыми:

```text
combat
anchor_path_closed
```

Стратегические враги статистику убийств не изменяют.

Убийства принимаются только в состоянии `RUNNING`.

### Финальный снимок

При `GameFlowController.run_ended` создаётся `RunStatisticsSnapshot`:

```text
survival_seconds
physical_kills
remaining_coins
purchased_upgrades
end_reason
```

После перехода в `GAME_OVER` снимок не меняется.

---

## 2. Рекорды текущего запуска приложения

`SessionRecordsStore` хранит:

```text
completed_runs
best_survival_seconds
best_physical_kills
```

Это узкое статическое хранилище:

- переживает `reload_current_scene()`;
- не является Node или autoload;
- не управляет игровой логикой;
- не сохраняет данные на диск;
- сбрасывается при полном перезапуске приложения.

После каждого завершённого забега `RunStatistics` регистрирует финальный снимок в `SessionRecordsStore` до показа итогового экрана.

---

## 3. Итоговый экран

Компоненты:

```text
GameOverPanel
scenes/ui/game_over_panel.tscn
```

Экран показывает:

- причину поражения;
- время выживания;
- количество физических убийств;
- остаток монет;
- число купленных карточек;
- количество завершённых забегов текущего запуска;
- лучшее время;
- лучший результат по убийствам.

Причины:

```text
shield_section_destroyed → секция щита разрушена
all_defenders_dead → экипаж уничтожен
```

Новый забег запускается:

- кнопкой `НОВЫЙ ЗАБЕГ`;
- действием `ui_accept` (`Enter`).

`GameFlowController.restart_run()` перезагружает текущую сцену, поэтому новый забег получает чистое состояние всех систем.

---

## 4. Текущая статистика в HUD

HUD показывает:

```text
Результат: MM:SS | физические убийства N
```

HUD только читает `RunStatistics` и не хранит собственный счётчик или таймер.

---

## 5. Критическое предупреждение щита

Компоненты:

```text
ShieldCriticalAlertController
ShieldCriticalAlertPresenter
scenes/ui/shield_critical_alert_presenter.tscn
```

### Controller

Подписывается на:

```text
ShieldSystem.section_entered_critical
```

Правила:

- предупреждение создаётся только в `RUNNING`;
- секции, вошедшие в критическое состояние в одном кадре, собираются вместе;
- выдаётся один сигнал `critical_alert_raised(section_ids)`;
- при выходе из `RUNNING` ожидающее предупреждение очищается;
- повторного сигнала нет, пока секция не выйдет и снова не войдёт в критический диапазон.

### Presenter

Показывает сообщение вида:

```text
КРИТИЧЕСКИЙ ЩИТ: S1, S2
```

И воспроизводит один короткий процедурный PCM-сигнал:

```text
Частота: 880 Гц
Длительность: 0.18 секунды
Амплитуда: 0.35
```

Аудиопоток создаётся один раз как `AudioStreamWAV` из 16-битных PCM-сэмплов. Presenter не читает и не изменяет прочность щита.

---

## 6. Общая сложность

`RunDifficulty` остаётся единственным владельцем активного времени и нормализованной сложности.

```text
Время до максимальной сложности: 600 секунд
```

Сложность растёт только в `RUNNING`.

Физическое давление:

```text
Наземный лимит: 8 → 20
Интервал спавна: 3.0 → 0.8 секунды
```

Стратегическое давление:

```text
Интервал волн: 12 → 4 секунды
Размер волны: 6 → 30
Время движения: 8 → 4 секунды
Цели: 1 → 3 секции
```

---

## 7. Стратегические массы

Реализованы:

- агрегированные группы без физических Node;
- реальное количество врагов;
- движение к секциям;
- отдельный импульс урона от каждого врага;
- объединение близких масс;
- разделение на 2–3 части;
- перенаправление частей;
- сохранение общего количества врагов;
- плавные маршруты без телепортации;
- постоянно видимая миникарта;
- отсутствие монет и физических убийств за стратегических врагов.

---

## 8. Физический абордаж и экипаж

Реализованы:

- спавн только при наличии якоря;
- общий лимит наземных врагов;
- движение по земле, тросам и платформе;
- очереди и разделение;
- взаимное блокирование;
- прыжок через защитника;
- ближний бой;
- три защитника;
- роли и физические переходы;
- независимые кулдауны замен;
- поражение при отсутствии живого экипажа.

---

## 9. Экономика и карточки

`RunEconomy` владеет монетами.

```text
Стартовые монеты: 0
Награда за физического врага: 1
```

Карточки:

```text
5, 10, 15, …, 100, 200, 400, 800
```

Обе карточки одинаковы и пока не применяют эффект.

## 10. Новые тесты

```text
tests/integration/run_statistics_game_over_scenarios.gd
tests/integration/shield_critical_alert_scenarios.gd
```

Проверяется:

- два награждаемых физических события дают два убийства;
- время берётся из `RunDifficulty`;
- финальный снимок содержит время, убийства, монеты, покупки и причину;
- после `GAME_OVER` убийства не увеличиваются;
- новый забег очищает текущую статистику;
- рекорды сохраняются между двумя забегами;
- итоговый экран появляется и показывает формат `MM:SS`;
- две критические секции одного кадра дают один сигнал;
- повторный вход секции создаёт новое предупреждение;
- во время карточек предупреждение не создаётся;
- Presenter получает `AudioStreamWAV`.

Команды:

```bash
godot --headless --path . --script res://tests/integration/run_statistics_game_over_scenarios.gd
godot --headless --path . --script res://tests/integration/shield_critical_alert_scenarios.gd
python tools/check_file_sizes.py
```

В среде помощника нет исполняемого Godot, поэтому headless-тесты здесь не запускались.

## 11. Временные ограничения

Пока не реализованы:

- постоянное сохранение рекордов между запусками приложения;
- медицинский пост и лекарь;
- турели;
- разные карточки и эффекты;
- финальная графика и анимации;
- полноценное звуковое оформление.
