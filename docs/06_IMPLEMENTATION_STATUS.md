# GloryProtect — статус реализации

## Целевая версия движка

Проект разрабатывается на **Godot 4.6.2 stable** с использованием GDScript.

## Текущая версия

**Prototype 0.4 — data-driven платформа и первый физический экипаж.**

---

## 1. Каркас забега

Реализованы:

- `project.godot` и главная сцена;
- стартовая безопасная задержка;
- состояния активной игры, ручной паузы, карточек и поражения;
- полная остановка симуляции во время паузы;
- корректное восстановление предыдущего состояния.

---

## 2. PlatformDomain

### Движение

Реализованы:

- горизонтальное движение через силу;
- инерция и сопротивление;
- максимальная скорость;
- мировые границы;
- якорные ограничения;
- обнуление скорости при достижении ограничения;
- камера, следующая за платформой.

### Декомпозиция

`PlatformController` больше:

- не читает клавиатуру;
- не хранит параметры баланса внутри скрипта;
- не рисует платформу.

Компоненты:

```text
PlatformController
SteeringInputProvider
WindSystem
PlatformVisualController
```

### Data-driven ресурсы

```text
scripts/platform/platform_balance.gd
resources/balance/platform_balance.tres
scripts/platform/wind_balance.gd
resources/balance/wind_balance.tres
```

В `PlatformBalance` находятся:

- количество и ширина клеток;
- высота платформы;
- рулевая сила;
- сопротивление;
- максимальная скорость;
- границы мира;
- размеры и положение постоянных постов.

В `WindBalance` находятся:

- силы уровней 1–3;
- интервалы изменений;
- амплитуда колебаний;
- скорость колебаний.

## 3. AnchorDomain

Якорная система декомпозирована:

```text
AnchorSystem
├── AnchorRuntimeStore
├── AnchorCommandController
├── AnchorOperationQueue
├── AnchorConstraintProvider
├── AnchorOverloadController
├── AnchorGeometry
└── AnchorVisualController
```

Реализованы:

- четыре якоря;
- фиксированные точки крепления;
- расширенная зона установки;
- зелёные и серые силуэты;
- установка с задержкой;
- очередь второго якоря стороны;
- экстренное снятие;
- конечная длина троса в обе стороны;
- направленное удержание;
- полная фиксация с двух сторон;
- перегрузка одиночного натянутого якоря при ветре силы 3;
- ограниченное растяжение перегрузки;
- срыв и автоматический возврат;
- сброс перегрузки вторым якорем или ослаблением ветра.

Параметры находятся в:

```text
resources/balance/anchor_balance.tres
```

## 4. Исправление бесконечного троса

Каждый якорь теперь задаёт две границы:

- немедленную границу в удерживаемом направлении;
- геометрическую границу конечной длины в противоположном направлении.

Правый якорь:

- сразу блокирует движение влево;
- разрешает движение вправо только до конечной длины;
- при ветре силы 1–2 останавливает платформу;
- при ветре силы 3 перегружается и срывается.

Для левого якоря правило зеркальное.

## 5. CrewDomain

### Физический экипаж

На платформе создаются три отдельных защитника.

Каждый защитник состоит из:

```text
Defender
├── HealthComponent
├── DefenderMovement
└── DefenderVisual
```

У каждого:

- уникальный идентификатор;
- три сегмента здоровья;
- собственная позиция;
- отдельный компонент движения;
- временный визуал;
- сигнал смерти.

### Владельцы состояния

```text
CrewManager          — реестр защитников
CrewRoleManager      — назначения и переходы
RoleStationRegistry  — занятость рабочих постов
HealthComponent      — здоровье конкретной сущности
```

Роль не хранится внутри `Defender` и не дублируется в HUD.

### Начальные назначения

- защитник 1 — рулевой;
- защитник 2 — левый якорщик;
- защитник 3 — правый якорщик.

Защитники физически находятся у соответствующих постов.

### Переназначение

В текущем прототипе доступны:

- рулевой;
- левый якорщик;
- правый якорщик;
- свободный защитник.

Реализованы состояния:

```text
ACTIVE
WAITING_FOR_ACTION
MOVING
DEAD
```

Новая роль:

1. проверяется на доступность;
2. резервирует пост;
3. ждёт завершения неделимого действия;
4. отключает старый пост;
5. заставляет защитника физически двигаться;
6. активируется только после прибытия.

### Рулевой

- При активном вводе команда переназначения сохраняется.
- Рулевой остаётся у поста, пока удерживается хотя бы одна кнопка направления.
- После отпускания ввод отключается.
- Новый рулевой включает управление только после физического прибытия.

### Якорщики

- Якорщик не уходит во время текущей установки.
- После её завершения старый пост отключается.
- Не начатые команды очереди отменяются при потере оператора.
- Новый якорщик активирует сторону только после прибытия.

### Смерть

При достижении 0 здоровья защитник:

- прекращает движение;
- скрывается;
- освобождает текущий и зарезервированный пост;
- отключает связанную способность;
- переходит в состояние `DEAD`.

Таймеры замены пока не реализованы.

## 6. Временное управление экипажем

| Действие | Клавиша |
|---|---|
| Выбрать защитника 1 | `5` |
| Выбрать защитника 2 | `6` |
| Выбрать защитника 3 | `7` |
| Назначить рулевым | `D` |
| Назначить левым якорщиком | `Z` |
| Назначить правым якорщиком | `X` |
| Назначить свободным | `C` |

Это временный `CrewDebugInput`. Финальный интерфейс будет использовать клики по персонажам и карточкам.

## 8. Тесты

### Якоря

```text
tests/unit/anchor_constraint_scenarios.gd
```

Проверяет конечную длину, перегрузку в обе стороны и устойчивость пары якорей.

### Ресурсы платформы

```text
tests/unit/platform_resource_scenarios.gd
```

Проверяет получение размеров и сил ветра из ресурсов, а также блокировку ввода без рулевого.

### Роли экипажа

```text
tests/integration/crew_role_scenarios.gd
```

Проверяет:

- начальные назначения;
- ожидание отпускания рулевого ввода;
- отключение управления во время перехода;
- включение после прибытия нового рулевого;
- ожидание завершения якорной установки перед уходом якорщика.

Команды запуска:

```bash
godot --headless --path . --script res://tests/unit/anchor_constraint_scenarios.gd
godot --headless --path . --script res://tests/unit/platform_resource_scenarios.gd
godot --headless --path . --script res://tests/integration/crew_role_scenarios.gd
python tools/check_file_sizes.py
```

В рабочей среде помощника отсутствует исполняемый файл Godot, поэтому Godot-тесты добавлены, но здесь не запускались.

## 9. Временные ограничения

- Используется один тестовый шар вместо пяти.
- Используется одна тестовая секция щита.
- Снятие отдельного якоря пока мгновенное.
- Свободный защитник ещё не ищет врагов.
- Ближний бой ещё не реализован.
- Лекарь и турельщик ещё недоступны.
- Таймеры замены погибших отсутствуют.
- Физические абордажные враги отсутствуют.
- Финальное контекстное меню ролей отсутствует.
- Звуковые эффекты отклонённых команд отсутствуют.
