# GloryProtect — статус реализации

## Целевая версия

Godot **4.6.2 stable**, строго типизированный GDScript.

## Текущая версия

**Prototype 1.4 — объединение, разделение и перенаправление стратегических масс.**

---

## 1. Забег и общая сложность

Реализованы:

- безопасная стартовая задержка;
- активная игра;
- полная ручная пауза;
- полная пауза выбора карточек;
- завершение забега;
- поражение при 0% любой секции;
- поражение при отсутствии живых защитников;
- единый `RunDifficulty`;
- сброс экономики, карточек, сложности, волн и мутаций при новом забеге.

Сложность растёт только в `RUNNING`.

```text
Время до максимальной сложности: 600 секунд
Нормализованная сложность: 0.0 → 1.0
```

---

## 2. Стратегический домен

Компоненты:

```text
StrategicWaveBalance
StrategicWaveDirector
StrategicWaveSystem
StrategicGroupMutationController
StrategicGroupRuntime
StrategicGroupSnapshot
```

### StrategicWaveDirector

Владеет только:

- номером волны;
- таймером следующей волны;
- RNG распределения новой волны.

Он выбирает уникальные секции, распределяет реальное количество врагов и создаёт группы через `StrategicWaveSystem`.

### StrategicWaveSystem

Является единственным владельцем массива активных групп.

Он выполняет только атомарные изменения состояния:

- создание;
- движение;
- переход к ударам;
- нанесение урона через `ShieldSystem`;
- удаление исчерпанной группы;
- объединение двух групп;
- разделение одной группы.

### StrategicGroupMutationController

Не хранит группы. Раз в заданный интервал он:

1. читает `StrategicGroupSnapshot`;
2. ищет ближайшую допустимую пару для объединения;
3. если объединение невозможно, проверяет случайное разделение;
4. отправляет одну команду владельцу `StrategicWaveSystem`;
5. не выполняет больше одной мутации за проверку.

Мутации выполняются только в `RUNNING`.

## 3. Положение и маршруты групп

`StrategicGroupRuntime` теперь хранит доменное положение на стратегической карте:

```text
map_angle
map_distance
route_start_angle
route_start_distance
route_target_angle
route_elapsed
travel_duration
```

`map_distance`:

```text
1.0 — внешний край миникарты
0.0 — поверхность щита
```

Положение принадлежит симуляции, а не миникарте. Это позволяет перенаправлять группу из её текущей точки без визуального телепорта.

`StrategicGroupSnapshot` публикует:

```text
group_id
section_id
enemy_count
initial_enemy_count
progress
map_angle
map_distance
is_impacting
mutation_ready
```

Изменяемый runtime наружу не выдаётся.

## 4. Объединение стратегических масс

Две группы могут объединиться, если:

- обе находятся в `TRAVELING`;
- закончился их кулдаун мутации;
- угловое расстояние меньше допуска;
- радиальное расстояние меньше допуска.

Текущие допуски:

```text
Угол: 0.16 радиана
Расстояние до щита: 0.08
```

При объединении:

- более крупная группа становится выжившей;
- цель более крупной группы сохраняется;
- количество врагов суммируется;
- `initial_enemy_count` суммируется;
- положение вычисляется как взвешенное среднее;
- оставшееся время маршрута вычисляется как взвешенное среднее;
- поглощённая группа удаляется;
- общее количество стратегических врагов не меняется.

Если группы равны, сохраняется цель первой выбранной группы.

---

## 5. Разделение и перенаправление

Группа может разделиться, если:

- она находится в `TRAVELING`;
- закончился кулдаун мутации;
- количество врагов не меньше 10;
- прогресс находится между 15% и 75%;
- есть свободные слоты групп;
- случайная проверка успешна.

Текущая вероятность за проверку:

```text
Начало сложности: 4%
Максимальная сложность: 22%
```

Разделение создаёт 2–3 части.

Правила:

- каждая часть получает минимум одного врага;
- остальные враги распределяются случайно;
- сумма частей строго равна исходной группе;
- все части стартуют из одной `map_angle/map_distance`;
- первая часть сохраняет прежнюю секцию;
- каждая дополнительная часть с вероятностью 65% получает другую секцию;
- при возможности перенаправленные части выбирают разные секции;
- маршрут каждой части перестраивается из точки разделения;
- после мутации действует кулдаун 2 секунды.

## 6. Стратегические волны и удары

Текущие параметры:

```text
Первая волна: 5 секунд после начала RUNNING
Интервал волн: 12 → 4 секунды
Размер волны: 6 → 30 врагов
Время движения: 8 → 4 секунды
Атакуемые секции: 1 → 3
Урон врага: 1
Интервал ударов: 0.35 секунды
Максимум активных групп: 15
```

После прибытия группа переходит в `IMPACTING`:

1. один враг наносит один импульс урона;
2. `enemy_count` уменьшается на единицу;
3. миникарта уменьшает массу;
4. после последнего врага группа удаляется.

Урон проходит только через:

```text
ShieldSystem.apply_damage(section_id, amount)
```

---

## 7. Агрегированная модель и экономика

Стратегические враги:

- не создаются как `Node2D`;
- не имеют физических боевых компонентов;
- не взаимодействуют с защитниками;
- не используют якоря;
- не попадают в `BoardingEnemyRegistry`;
- не дают монет.

`RunEconomy` остаётся единственным владельцем валюты. Награда выдаётся только за физические причины:

```text
combat
anchor_path_closed
```

---

## 8. Стратегическая миникарта

Миникарта всегда видима в правом нижнем углу.

Она читает доменные `map_angle` и `map_distance`, поэтому показывает:

- реальное сближение масс;
- увеличение одной массы после объединения;
- появление частей в одной точке;
- плавное расхождение к новым секциям;
- реальное количество врагов;
- сокращение массы при ударах;
- проценты и состояние пяти секций;
- номер волны и время до следующей.

Миникарта получает только снимки и не изменяет симуляцию.

---

## 9. Физический абордаж

Реализованы:

- отсутствие спавна без якорей;
- общий наземный лимит `8 → 20`;
- интервал спавна `3.0 → 0.8` секунды;
- выбор стороны 50/50;
- движение по земле и тросам;
- очереди и разделение;
- взаимное блокирование;
- прыжок через защитника;
- ближний бой;
- награда только за физические смерти.

Физический и стратегический спавнеры независимы, но читают один `RunDifficulty`.

---

## 10. Карточки

`UpgradeSystem` показывает две одинаковые карточки-заглушки.

```text
Стоимость: 5, 10, 15, …, 100, 200, 400, 800
```

Карточки не применяют эффект. Выбор только списывает стоимость и увеличивает счётчик покупок.

## 12. Временные ограничения

Пока не реализованы:

- звуковой сигнал новой критической секции;
- статистика времени выживания и физических убийств;
- разные карточки и эффекты;
- лекарь и турели;
- финальная графика миникарты;
- финальные анимации и звук.
