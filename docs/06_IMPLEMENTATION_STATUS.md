# GloryProtect — статус реализации

## Целевая версия

Godot **4.6.2 stable**, строго типизированный GDScript.

## Текущая версия

**Prototype 1.9 — мышиный интерфейс выбора экипажа и назначения ролей.**

---

## 1. Единый выбор защитника

Добавлен:

```text
scripts/crew/crew_selection_controller.gd
```

`CrewSelectionController` является единственным владельцем:

```text
selected_defender_id
```

Выбор выполняется:

- клавишами `5`, `6`, `7`;
- кнопками защитников в UI;
- левым кликом по персонажу на платформе.

Старый `CrewDebugInput` сохранён как совместимая оболочка:

```text
CrewDebugInput extends CrewSelectionController
```

Поэтому существующие `NodePath` и debug-команды не сломаны, но второго состояния выбора больше нет.

### Мировой клик

`CrewSelectionController` получает необработанный клик после UI, преобразует экранную координату через canvas transform и выбирает ближайшего живого защитника в пределах его визуального радиуса.

Клик по кнопке интерфейса не проходит в мировой выбор.

### Подсветка

`DefenderVisual` получил:

```text
set_selected(is_selected)
is_selected()
```

Выбранный защитник отмечается жёлтым кольцом. После замены персонажа тот же стабильный `defender_id` остаётся выбранным.

## 2. CrewCommandPanel

Добавлены:

```text
scripts/ui/crew_command_panel.gd
scripts/ui/crew_command_panel_view.gd
scripts/ui/crew_command_text.gd
```

`PrototypeHUD` создаёт панель программно и передаёт ей существующие зависимости.

Панель отображает:

- всех защитников;
- здоровье каждого;
- текущую роль;
- целевую роль во время перехода;
- таймер замены после смерти;
- выбранного защитника;
- стандартные роли;
- медицинский пост;
- все установленные турели;
- клетку каждой турели;
- владельца каждого поста;
- понятную причину отказа команды.

---

## 3. Разделение UI

### CrewCommandPanel

Координирует чтение состояния и отправку команд:

```text
CrewSelectionController
CrewManager
CrewRoleManager
CrewReplacementController
BuildableGrid
```

Он не изменяет назначение напрямую и вызывает только:

```text
CrewRoleManager.request_assignment(...)
```

### CrewCommandPanelView

Создаёт и хранит только контролы:

- портрет-кнопки;
- кнопки ролей;
- кнопки турелей;
- строки состояния и обратной связи.

### CrewCommandText

Форматирует названия ролей, занятость постов, переходы и причины отказа. Не хранит состояние и не обращается к дереву сцены.

`crew_command_panel.gd` после разделения содержит около 336 строк и не приближается к лимиту 600.

---

## 4. Доступность команд

Панель предварительно отключает кнопку, если:

- мировая симуляция не активна;
- защитник погиб;
- защитник уже движется к другой роли;
- защитник ожидает завершения лечения или выстрела;
- пост отсутствует;
- пост занят другим защитником;
- выбранная роль уже является текущей на том же посту.

Окончательную проверку продолжает выполнять `CrewRoleManager`.

Синхронные причины отказа:

```text
unknown_defender
defender_dead
defender_busy
role_unavailable
station_occupied
```

переводятся в понятный текст в нижней строке панели.

---

## 5. Стандартные роли

В панели доступны:

```text
Свободный боец
Рулевой
Левый якорщик
Правый якорщик
Лекарь
```

Лекарь отображается всегда, но кнопка остаётся недоступной до установки медицинского поста.

Текущая роль выбранного защитника блокируется как уже активная.

---

## 6. Конкретные турели

Каждая установленная турель создаёт отдельную кнопку:

```text
Турель T2 • клетка 15
занято защитником 3
```

Кнопка отправляет:

```text
request_assignment(
    selected_defender_id,
    CrewRole.Id.TURRET,
    buildable_id
)
```

Если пост занят другим защитником, кнопка отключена. Перенос обновляет номер клетки без пересоздания доменного состояния. Демонтаж удаляет кнопку.

---

## 7. Пауза и диагностический HUD

### CARD_SELECTION

- панель экипажа скрывается;
- команды недоступны;
- мышь не выбирает персонажей в мире.

### MANUAL_PAUSE

- панель остаётся видимой;
- все кнопки назначения отключены;
- состояние экипажа продолжает отображаться.

Большая диагностическая панель теперь скрыта по умолчанию. Она переключается клавишей:

```text
F10
```

Обычная игра больше не требует видеть длинную таблицу debug-телеметрии.

---

## 8. Совместимость клавиатуры

Сохранены:

```text
5 / 6 / 7  выбрать защитника
D          рулевой
Z / X      левый / правый якорщик
C          свободный боец
H          лекарь
J          оператор выбранной турели
```

`BuildableDebugInput` и `TurretDebugInput` читают выбранного защитника только через:

```text
get_selected_defender_id()
```

## 10. Новый интеграционный тест

```text
tests/integration/crew_command_panel_scenarios.gd
```

Проверяется:

- создание кнопок трёх защитников;
- единый выбранный ID;
- визуальное кольцо выбранного защитника;
- смена выбора через интерфейсный API;
- назначение свободного бойца;
- недоступность лекаря без поста;
- появление лекаря после установки поста;
- появление кнопки конкретной турели;
- назначение оператора;
- блокировка занятой турели для другого защитника;
- скрытие панели во время карточек;
- отключение команд во время карточек;
- восстановление панели после закрытия карточек.

Команды:

```bash
godot --headless --path . --script res://tests/integration/crew_command_panel_scenarios.gd
godot --headless --path . --script res://tests/integration/turret_scenarios.gd
godot --headless --path . --script res://tests/integration/buildable_medical_station_scenarios.gd
python tools/check_file_sizes.py
```

В среде помощника нет исполняемого Godot, поэтому headless-тесты здесь не запускались.

## 12. Временные ограничения

Пока не реализованы:

- настоящие спрайты защитников и врагов;
- покадровые анимации idle, бега, атаки и смерти;
- финальные ассеты турели и медицинского поста;
- полноценный интерфейс установки и переноса объектов мышью;
- постоянное сохранение рекордов;
- разные карточки улучшений.
