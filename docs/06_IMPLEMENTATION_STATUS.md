# GloryProtect — статус реализации

## Целевая версия движка

Проект разрабатывается на **Godot 4.6.2 stable** с использованием GDScript.

## Текущая версия прототипа

**Prototype 0.2 — движение, энергетический контакт и якоря.**

Реализованы этапы базового каркаса, движения и первая версия центральной якорной механики.

### Каркас забега

- создан `project.godot`;
- назначена главная сцена;
- добавлен контроллер состояния забега;
- добавлена стартовая безопасная задержка;
- добавлена полная ручная пауза;
- пауза во время стартовой задержки корректно возвращает предыдущее состояние.

### Платформа и ветер

- реализован ветер направлений влево и вправо;
- реализованы три уровня силы ветра;
- добавлены небольшие колебания фактической силы;
- реализовано физическое горизонтальное управление платформой;
- управление работает только при назначенном рулевом;
- добавлены инерция, сопротивление и ограничение скорости;
- добавлены невидимые границы тестового мира;
- камера следует за платформой.

### Энергетический контакт

- добавлен один тестовый наземный шар;
- добавлена узкая зона энергетического контакта;
- добавлена тестовая секция щита;
- контакт восстанавливает секцию с постоянной скоростью;
- временная команда позволяет наносить тестовый урон секции.

### Якоря

- добавлены четыре якоря: два левых и два правых;
- добавлена расширенная зона установки;
- точки крепления скрыты вне зоны;
- в зоне показываются временные цветные силуэты;
- для каждого якоря используется отдельная клавиша;
- одна клавиша переключает установку и снятие;
- установка занимает время;
- второй якорь одной стороны можно поставить в очередь;
- очередь отменяется, если к началу следующей установки команда больше недопустима;
- добавлена команда экстренного снятия всех якорей;
- начатая установка завершается до выполнения экстренного снятия;
- реализована максимальная длина троса;
- недостаточная длина при завершении установки вызывает немедленный обрыв;
- сорванный якорь автоматически возвращается;
- левые якоря ограничивают движение вправо;
- правые якоря ограничивают движение влево;
- якоря с обеих сторон полностью фиксируют платформу;
- при достижении предела троса скорость платформы обнуляется;
- маятникового движения нет;
- одиночный якорь перегружается при ветре силы 3 в блокируемом направлении;
- перегрузка длится примерно 2,5 секунды;
- во время перегрузки трос мигает красным;
- второй якорь той же стороны отменяет перегрузку;
- ослабление или смена направления ветра отменяет перегрузку;
- после обрыва якорь возвращается на платформу.

### Диагностика

HUD показывает:

- состояние забега;
- направление, уровень и фактическую силу ветра;
- координату и скорость платформы;
- состояние каждого якоря;
- доступность зоны установки;
- наличие энергетического контакта;
- прочность тестовой секции.

Временная графика рисуется кодом и не требует внешних ассетов.

## Запуск

1. Установить Godot 4.6.2 stable.
2. Клонировать репозиторий.
3. Открыть `project.godot` через Project Manager.
4. Запустить проект клавишей `F5`.

Главная сцена:

```text
res://scenes/game/game_root.tscn
```

## Управление Prototype 0.2

| Действие | Управление |
|---|---|
| Приложить силу влево | стрелка влево |
| Приложить силу вправо | стрелка вправо |
| Установить или снять якорь 1 | `1` |
| Установить или снять якорь 2 | `2` |
| Установить или снять якорь 3 | `3` |
| Установить или снять якорь 4 | `4` |
| Экстренно снять все якоря | `R` |
| Нанести тестовой секции 10 урона | `Space` |
| Включить или снять ручную паузу | `Esc` |

`Space` является только временным инструментом тестирования. В финальной игре урон секциям наносят стратегические враги.

## Временные ограничения прототипа

- Якорщики пока представлены булевым состоянием доступности, а не физическими защитниками.
- Снятие отдельного якоря пока выполняется мгновенно после проверки наличия оператора.
- Враги на тросах ещё не реализованы.
- Используется один тестовый шар вместо пяти.
- Используется одна тестовая секция щита.
- Балансовые параметры пока заданы экспортируемыми полями скриптов, а не отдельными `.tres`-ресурсами.
- Звуковой эффект недоступного действия ещё не подключён.

## Проверка

Все текущие GDScript-файлы, включая якорную систему, проверены статическим линтером `gdlint 4.5.0`: синтаксических проблем не обнаружено.

Полноценный импорт и запуск через Godot 4.6.2 необходимо дополнительно проверить после клонирования. В рабочей среде помощника отсутствует исполняемый файл Godot, а автоматическая загрузка бинарника была заблокирована окружением.
