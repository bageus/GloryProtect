# GloryProtect — статус реализации

## Целевая версия

Godot **4.6.2 stable**, строго типизированный GDScript.

## Текущая версия

**Prototype 1.7 — несколько размещаемых турелей и отдельные операторы.**

---

## 1. BuildableDomain

Реализованы два типа объектов:

```text
MEDICAL_STATION
TURRET
```

`BuildableInventory` владеет открытым количеством:

```text
MEDICAL_STATION: максимум 1
TURRET: максимум 4
```

`BuildableGrid` владеет размещением и занятостью клеток:

```text
place(type_id, cell_index)
move(buildable_id, cell_index)
demolish(buildable_id)
get_buildable_ids_by_type(type_id)
get_snapshots()
```

Правила:

- одна клетка содержит максимум один объект;
- служебные клетки недоступны;
- персонаж на клетке не блокирует установку;
- объекты не создают коллизий;
- перенос освобождает старую клетку;
- демонтаж освобождает клетку;
- открытие сохраняется после демонтажа.

Координаты клеток предоставляет только `PlatformController`.

## 2. Конкретные посты ролей

`CrewAssignmentRuntime` теперь хранит:

```text
current_role
current_station_id
target_role
target_station_id
state
```

Статические посты используют `station_id = -1`.

Медицинский пост использует:

```text
MEDIC / 0
```

Каждая турель использует:

```text
TURRET / buildable_id
```

Поэтому несколько турелей являются независимыми постами одной роли.

`RoleStationRegistry` хранит владельца и координату каждого составного ключа. Один пост нельзя занять двумя защитниками.

При переносе динамического поста:

- защитник, уже идущий к нему, получает новую цель;
- активный оператор турели физически переходит к новой позиции;
- если начато неделимое действие, переход ждёт его завершения.

## 3. TurretDomain

Компоненты:

```text
TurretRuntime
TurretTargetSelector
TurretSystem
TurretDebugInput
```

### TurretRuntime

Каждая турель имеет независимые:

```text
operator_id
target_enemy_id
shot_remaining
cooldown_remaining
firing
```

Клетка и размещение читаются из `BuildableGrid` и не дублируются.

### TargetSelector

Турель выбирает ближайшего живого физического врага в круговом радиусе.

Допустимые состояния:

```text
CLIMBING
ON_PLATFORM
FIGHTING
JUMPING
```

Наземные враги без пути не атакуются.

При равной дистанции используется меньший стабильный `enemy_id`.

### Стрельба

Текущий баланс:

```text
Урон: 1
Радиус: 360
Подготовка выстрела: 0.45 секунды
Кулдаун: 0.8 секунды
```

Последовательность:

```text
оператор прибыл
→ выбрать ближайшую цель
→ зафиксировать target_enemy_id
→ turret_shot_windup
→ HealthComponent.apply_damage(1)
→ turret_shot_cooldown
```

Начатый выстрел не перенаправляется. Если цель погибла раньше, выстрел завершается без попадания.

Несколько турелей могут одновременно стрелять в одного врага.

## 4. Операторы турелей

Каждая турель требует отдельного защитника.

Турель работает только если:

- её объект установлен;
- пост роли зарегистрирован;
- назначенный защитник жив;
- защитник физически прибыл;
- текущая роль и `station_id` совпадают с этой турелью.

Роль `TURRET` не использует ближний бой. Враг на позиции турели атакует самого оператора. Отдельного здоровья у турели нет.

### Переназначение

Во время подготовки выстрела `TurretSystem` поднимает общий флаг неделимого действия.

Если оператору назначена новая роль:

1. состояние переходит в `WAITING_FOR_ACTION`;
2. текущий выстрел завершается;
3. новый выстрел не начинается;
4. флаг действия снимается;
5. защитник физически идёт к новой роли.

### Перенос

Турель мгновенно меняет клетку в `BuildableGrid`.

- без активного выстрела оператор сразу начинает переход;
- во время выстрела сначала завершается выстрел;
- пока оператор не прибыл, турель не работает;
- назначение остаётся зарезервировано за тем же защитником.

### Демонтаж

При демонтаже:

- текущий выстрел отменяется;
- динамический пост удаляется;
- оператор немедленно становится свободным бойцом;
- runtime турели удаляется;
- открытие сохраняется.

---

## 5. Награды и статистика

`TurretSystem` не начисляет монеты напрямую.

Успешный смертельный выстрел проходит через существующую цепочку:

```text
HealthComponent.depleted
→ BoardingEnemy.kill("combat")
→ BoardingEnemyRegistry.enemy_removed
→ BoardingRewardController
→ RunEconomy
→ RunStatistics
```

Поэтому убийство турелью даёт ту же физическую награду и учитывается в статистике.

---

## 6. Медицинский пост

Система Prototype 1.6 сохранена:

- один медицинский пост;
- роль `MEDIC`;
- выбор наиболее раненого союзника;
- один сегмент за пять секунд;
- завершение текущего цикла перед переназначением;
- перенос без прерывания лечения;
- немедленное освобождение лекаря при демонтаже.

Медицинский пост и турели используют одну систему конкретных динамических постов.

## 7. Debug-управление Prototype 1.7

```text
, / .       выбрать клетку
T           открыть ещё одну турель
G           установить новую турель
K           выбрать следующую турель
V           перенести выбранную турель
J           назначить выбранного защитника оператором
Backspace   демонтировать выбранную турель
```

Остальные команды:

```text
5/6/7       выбрать защитника
C           свободный боец
H           лекарь
B/M/Delete  медицинский пост
```

Карточки остаются заглушками и не открывают объекты.

## 8. Визуал и HUD

`BuildableGridVisual` рисует:

- выбранную клетку;
- медицинский пост;
- все установленные турели;
- кольцо вокруг выбранной турели.

HUD показывает:

```text
открытое количество турелей
установленное количество
выбранную турель и клетку
оператора каждой турели
зафиксированную цель
оставшееся время выстрела
```

Визуал не владеет стрельбой и не создаёт коллизий.

## 9. Интеграционный тест

```text
tests/integration/turret_scenarios.gd
```

Проверяется:

- невозможность установки закрытой турели;
- открытие и установка двух турелей;
- два независимых поста роли;
- отдельный оператор каждой турели;
- запрет второго оператора на занятый пост;
- выбор ближайшей цели каждой турелью;
- ровно один урон за выстрел;
- обычные монеты и статистика за убийство;
- завершение выстрела перед переназначением;
- отсутствие автоматического перенаправления;
- перенос во время выстрела;
- физическое прибытие оператора к новой клетке;
- остановка работы во время перехода;
- демонтаж и немедленное освобождение роли;
- сохранение открытия после демонтажа.

Команды:

```bash
godot --headless --path . --script res://tests/integration/turret_scenarios.gd
godot --headless --path . --script res://tests/integration/buildable_medical_station_scenarios.gd
python tools/check_file_sizes.py
```

В среде помощника нет исполняемого Godot, поэтому headless-тесты здесь не запускались.

## 10. Временные ограничения

Пока не реализованы:

- финальная графика и анимация турели;
- видимый снаряд или линия выстрела;
- отображение радиуса;
- карточки, открывающие медицинский пост и турели;
- постоянное сохранение рекордов;
- полноценное звуковое оформление.

---

## 11. Следующая итерация

1. Анимация подготовки и выстрела турели.
2. Визуальный снаряд без изменения логики попадания.
3. Радиус выбранной турели.
4. Явное неактивное состояние без оператора.
5. Подготовка объектов к будущему каталогу карточек.

Карточки остаются заглушками без различных эффектов.
