# GloryProtect — статус реализации

## Целевая версия движка

Проект разрабатывается на **Godot 4.6.2 stable** с использованием GDScript.

## Текущая версия

**Prototype 0.5 — пять секций щита, пять шаров и общий реестр якорных точек.**

---

## 1. Каркас забега

Реализованы:

- стартовая безопасная задержка;
- активная игра;
- ручная пауза;
- пауза выбора карточек;
- состояние поражения;
- причина завершения забега;
- полная остановка симуляции во время паузы.

Поражение уже вызывается при:

- разрушении любой секции щита;
- отсутствии живых защитников.

---

## 2. PlatformDomain

Компоненты:

```text
PlatformController
SteeringInputProvider
WindSystem
PlatformVisualController
```

Реализованы:

- горизонтальное движение через силу;
- инерция и сопротивление;
- ограничение скорости;
- мировые и якорные границы;
- камера, следующая за платформой;
- управление только при активном рулевом;
- data-driven параметры платформы и ветра.

Ресурсы:

```text
resources/balance/platform_balance.tres
resources/balance/wind_balance.tres
```

---

## 3. OrbDomain

### Пять шаров

Создан ресурс:

```text
resources/definitions/ground_orb_catalog.tres
```

Он содержит:

- пять фиксированных горизонтальных позиций;
- высоту земли;
- ширину контактной зоны;
- размеры временного визуала;
- глубину земли.

Текущие позиции:

```text
-2000, -1000, 0, 1000, 2000
```

Платформа начинает у центрального, третьего шара.

### GroundOrbRegistry

Реестр предоставляет:

- координаты шаров;
- связь `orb_id → section_id`;
- поиск контактного шара;
- поиск шара в зоне установки якорей;
- четыре наземные точки крепления каждого шара.

Геометрия шаров больше не дублируется в `PrototypeWorld` или `AnchorSystem`.

### OrbContactSystem

Реализованы:

- выбор ближайшего шара внутри узкого допуска;
- немедленный разрыв после выхода из зоны;
- сигналы начала и завершения контакта;
- связь активного шара с соответствующей секцией.

Контакт не изменяет щит напрямую.

### GroundOrbVisualController

Отображает:

- непрерывную землю;
- пять шаров;
- индивидуальные цвета секций;
- кольца текущей прочности;
- тусклое состояние выше 50%;
- яркое состояние при 50% и ниже;
- мигание критического шара;
- энергетический луч активного контакта.

## 4. ShieldDomain

Компоненты:

```text
ShieldSystem
ShieldRechargeController
ShieldFailureController
ShieldDebugInput
```

### ShieldSystem

Является единственным владельцем пяти значений прочности.

Реализованы:

- пять секций по 100% в начале;
- независимый урон каждой секции;
- независимое восстановление;
- ограничение диапазона 0–100%;
- порог указателя 50%;
- критический порог 25%;
- события входа и выхода из критического состояния;
- событие разрушения секции.

Ресурс:

```text
resources/balance/shield_balance.tres
```

В него вынесены:

- число секций;
- максимальная прочность;
- скорость зарядки;
- пороги интерфейса;
- размер тестового урона;
- индивидуальные цвета секций.

### ShieldRechargeController

При активном контакте непрерывно восстанавливает только связанную секцию с постоянной скоростью.

### ShieldFailureController

При достижении 0% любой секцией завершает забег с причиной:

```text
shield_section_destroyed
```

### ShieldDebugInput

Временное управление:

- `F1–F5` — выбрать секцию;
- `Space` — нанести выбранной секции тестовый урон.

## 5. AnchorDomain

Композиция:

```text
AnchorSystem
├── AnchorRuntimeStore
├── AnchorCommandController
├── AnchorOperationQueue
├── AnchorConstraintProvider
├── AnchorOverloadController
├── AnchorGeometry
└── AnchorVisualController
```

Сохранены ранее реализованные механики:

- четыре якоря;
- отдельные команды;
- очередь второго якоря стороны;
- конечная длина троса;
- направленное удержание;
- полная фиксация;
- перегрузка одиночного троса;
- срыв и возврат.

### Интеграция с пятью шарами

Теперь каждая установка фиксирует:

- конкретный `orb_id`;
- конкретную наземную точку;
- позицию платформы в момент закрепления.

Установленный якорь не перепрыгивает к следующему шару при движении платформы.

Очередь второго якоря сохраняет исходный шар. Если до начала второй операции платформа вышла из его зоны, команда отменяется.

Силуэты появляются возле любого из пяти шаров.

## 6. CrewDomain

На платформе находятся три физических защитника.

Компоненты:

```text
CrewManager
CrewRoleManager
RoleStationRegistry
CrewFailureController

Defender
├── HealthComponent
├── DefenderMovement
└── DefenderVisual
```

Реализованы:

- стартовые роли рулевого и двух якорщиков;
- физическое перемещение между постами;
- резервирование рабочих мест;
- ожидание завершения рулевого ввода;
- ожидание завершения текущей установки якоря;
- активация новой роли после прибытия;
- три сегмента здоровья;
- смерть и освобождение поста;
- поражение при нуле живых защитников.

Таймеры замены пока не реализованы.

---

## 7. Диагностический HUD

HUD показывает:

- состояние забега;
- направление и силу ветра;
- позицию и скорость платформы;
- четыре состояния якорей;
- шар, возле которого доступны крепления;
- роли и переходы экипажа;
- активный энергетический контакт;
- проценты всех пяти секций;
- выбранную тестовую секцию;
- направления ко всем секциям с 50% или ниже;
- причину поражения.

## 8. Управление

### Платформа и якоря

| Действие | Клавиша |
|---|---|
| Рулевое усилие | `←` / `→` |
| Якоря | `1–4` |
| Снять все якоря | `R` |
| Пауза | `Esc` |

### Экипаж

| Действие | Клавиша |
|---|---|
| Выбрать защитника | `5`, `6`, `7` |
| Рулевой | `D` |
| Левый якорщик | `Z` |
| Правый якорщик | `X` |
| Свободный защитник | `C` |

### Щит

| Действие | Клавиша |
|---|---|
| Выбрать секцию | `F1–F5` |
| Нанести тестовый урон | `Space` |

## 9. Тесты

### Якорная геометрия

```text
tests/unit/anchor_constraint_scenarios.gd
```

Проверяет:

- конечную длину;
- перегрузку в обе стороны;
- устойчивость пары якорей;
- сохранение исходного `orb_id` после движения к другому шару.

### Ресурсы платформы

```text
tests/unit/platform_resource_scenarios.gd
```

Проверяет data-driven размеры, силы ветра и отсутствие управления без рулевого.

### Роли и поражение экипажа

```text
tests/integration/crew_role_scenarios.gd
```

Проверяет:

- стартовые назначения;
- отложенный уход рулевого;
- физическое прибытие нового рулевого;
- завершение установки перед уходом якорщика;
- поражение при смерти всех защитников.

### Пять шаров и секций

```text
tests/integration/shield_orb_scenarios.gd
```

Проверяет:

- создание пяти секций со 100%;
- выбор правильного шара;
- зарядку только связанной секции;
- отсутствие зарядки вне контакта;
- порог указателя 50%;
- поражение при разрушении секции.

Команды:

```bash
godot --headless --path . --script res://tests/unit/anchor_constraint_scenarios.gd
godot --headless --path . --script res://tests/unit/platform_resource_scenarios.gd
godot --headless --path . --script res://tests/integration/crew_role_scenarios.gd
godot --headless --path . --script res://tests/integration/shield_orb_scenarios.gd
python tools/check_file_sizes.py
```

В среде помощника отсутствует исполняемый файл Godot, поэтому headless-тесты добавлены, но фактически здесь не запущены.
