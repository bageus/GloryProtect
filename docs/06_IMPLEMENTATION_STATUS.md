# GloryProtect — статус реализации

## Целевая версия движка

Проект разрабатывается на **Godot 4.6.2 stable** с использованием GDScript.

## Текущая версия

**Prototype 0.3 — декомпозированная якорная система и конечная длина троса.**

---

## 1. Каркас забега

Реализованы:

- `project.godot` и главная сцена;
- состояния запуска, стартовой задержки, активной игры, паузы и поражения;
- безопасная задержка перед угрозами;
- полная ручная пауза;
- корректное возвращение к предыдущему состоянию после паузы.

---

## 2. Платформа и ветер

Реализованы:

- горизонтальное движение через силу;
- инерция и сопротивление;
- максимальная скорость;
- границы тестового мира;
- камера, следующая за платформой;
- ветер влево и вправо;
- уровни силы 1–3;
- небольшие колебания фактического усилия;
- управление только при доступном рулевом.

---

## 3. Энергетический контакт

Реализованы:

- один тестовый наземный шар;
- узкая зона контакта;
- тестовая секция щита;
- постоянная скорость зарядки внутри зоны;
- временная команда нанесения тестового урона.

---

## 4. Якорная система

Реализованы:

- четыре якоря;
- две левые и две правые точки;
- расширенная зона установки;
- скрытие силуэтов вне зоны;
- активные и недоступные силуэты;
- отдельные команды `1–4`;
- установка с длительностью;
- очередь второго якоря стороны;
- экстренное снятие всех якорей;
- завершение уже начатой установки перед экстренным снятием;
- немедленный обрыв при недостаточной длине в момент завершения установки;
- автоматический возврат сорванного якоря;
- направленное удержание;
- полная фиксация при якорях с обеих сторон;
- обнуление скорости при ограничении;
- отсутствие маятникового движения.

---

## 5. Исправление конечной длины троса

Исправлена ошибка, при которой правый якорь мог бесконечно растягиваться при движении платформы вправо. Зеркальная проблема существовала и для левого якоря при движении влево.

Теперь каждый установленный якорь задаёт две границы:

- немедленную границу в удерживаемом направлении;
- геометрическую границу конечной длины троса в противоположном направлении.

Пример для правого якоря:

- движение влево блокируется с позиции установки;
- движение вправо разрешено только до максимальной длины троса;
- после достижения предела скорость обнуляется;
- трос больше не растягивается бесконечно.

---

## 6. Исправление перегрузки

Перегрузка теперь определяется реальным натяжением троса.

Одиночный правый якорь при ветре силы 3:

- перегружается сразу, если ветер тянет влево;
- перегружается после достижения предела троса, если ветер тянет вправо.

Для левого якоря правило работает зеркально.

Во время перегрузки допускается только небольшое настраиваемое растяжение. После завершения таймера якорь срывается и возвращается.

Ветер силы 1–2 не срывает якорь: платформа останавливается на конечной длине.

Два якоря одной стороны отменяют перегрузку.

## 7. Декомпозиция AnchorDomain

Старый монолитный `anchor_system.gd` разделён на компоненты:

```text
scripts/anchors/
├── anchor_balance.gd
├── anchor_command_controller.gd
├── anchor_constraint_provider.gd
├── anchor_geometry.gd
├── anchor_operation_queue.gd
├── anchor_overload_controller.gd
├── anchor_runtime.gd
├── anchor_runtime_store.gd
├── anchor_system.gd
└── anchor_visual_controller.gd
```

Ответственности:

- `AnchorRuntimeStore` — единственный владелец состояний якорей;
- `AnchorCommandController` — проверка и маршрутизация команд;
- `AnchorOperationQueue` — текущие и поставленные в очередь операции;
- `AnchorGeometry` — точки крепления и геометрия троса;
- `AnchorConstraintProvider` — допустимый диапазон движения;
- `AnchorOverloadController` — логика перегрузки и срыва;
- `AnchorVisualController` — тросы, силуэты и предупреждения;
- `AnchorSystem` — тонкий координатор.

Все файлы находятся значительно ниже лимита 600 строк.

## 8. Data-driven баланс якорей

Созданы:

```text
scripts/anchors/anchor_balance.gd
resources/balance/anchor_balance.tres
```

В ресурс вынесены:

- зона установки;
- длительность установки;
- длительность перегрузки;
- длительность возврата;
- длина троса;
- ограниченное растяжение перегрузки;
- допуск определения натяжения;
- позиции наземных креплений;
- отступы креплений платформы.

## 9. Регрессионный тест

Добавлен сценарий:

```text
tests/unit/anchor_constraint_scenarios.gd
```

Он проверяет:

1. конечную правую границу правого якоря;
2. перегрузку правого якоря при ветре силы 3 вправо после натяжения;
3. перегрузку того же якоря при ветре силы 3 влево;
4. устойчивость двух правых якорей.

Команда для запуска после установки Godot 4.6.2:

```bash
godot --headless --path . --script res://tests/unit/anchor_constraint_scenarios.gd
```

В текущей рабочей среде отсутствует исполняемый файл Godot, поэтому тест добавлен, но его фактический headless-запуск здесь не выполнен.

## 10. Управление

| Действие | Управление |
|---|---|
| Приложить силу влево | стрелка влево |
| Приложить силу вправо | стрелка вправо |
| Установить или снять якорь 1 | `1` |
| Установить или снять якорь 2 | `2` |
| Установить или снять якорь 3 | `3` |
| Установить или снять якорь 4 | `4` |
| Экстренно снять все якоря | `R` |
| Нанести тестовой секции 10 урона | `Space` |
| Включить или снять паузу | `Esc` |

## 11. Временные ограничения

- Якорщики пока представлены булевым состоянием доступности.
- Снятие отдельного якоря пока мгновенное.
- Враги на тросах ещё не реализованы.
- Используется один тестовый шар вместо пяти.
- Используется одна тестовая секция.
- Параметры платформы и ветра ещё не вынесены в отдельные ресурсы.
- Звуковой эффект отклонённой команды ещё не подключён.
