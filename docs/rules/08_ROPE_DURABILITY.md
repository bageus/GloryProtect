# Правила: прочность тросов

Этот файл является источником правды для независимой прочности четырёх тросов, принятия урона и восстановления прочности при повторной установке.

Физика движения, перегрузка ветром, установка и снятие якорей остаются в [`02_PLATFORM_WIND_AND_ANCHORS.md`](02_PLATFORM_WIND_AND_ANCHORS.md).

---

## 1. Независимые шкалы

У каждого из четырёх якорей есть собственная шкала прочности троса.

- Повреждение одного троса не изменяет остальные.
- Максимальная прочность задаётся балансным ресурсом.
- Текущее значение ограничено диапазоном от `0` до максимума.
- UI получает read-only snapshot и не хранит копию gameplay-состояния.

---

## 2. Когда трос принимает урон

Урон может получить только трос якоря, который находится в состоянии:

- `ATTACHED`;
- `OVERLOADED`.

Снятый, возвращающийся, устанавливаемый или ожидающий установки якорь урон тросу не принимает.

Нулевой и отрицательный урон отклоняется.

---

## 3. Повреждение и перегрузка

Прочность троса и перегрузка ветром являются разными системами.

- Перегрузка запускается реальным натяжением при ветре силы `3` по правилам файла 02.
- Повреждение уменьшает отдельную шкалу прочности.
- Высокое или низкое значение прочности само по себе не запускает и не отменяет перегрузку.
- Отмена перегрузки не восстанавливает потерянную прочность.

---

## 4. Достижение нуля

Когда прочность впервые достигает `0`, `AnchorBreakRecoveryController` атомарно:

1. отменяет ожидающие операции именно этого якоря;
2. переводит установленный или перегруженный якорь в `RETURNING`;
3. немедленно закрывает соответствующий boarding-путь;
4. удаляет через общий death pipeline всех врагов, которые ещё находятся на этом тросе;
5. не затрагивает врагов, уже вышедших на платформу;
6. пересчитывает ограничения движения платформы;
7. публикует событие начала восстановления.

Враги на тросе погибают с причиной `anchor_path_closed`, поэтому используют существующий поток награды за снятие или обрыв якоря.

Повторный урон разрушенному тросу не создаёт второе событие.

Возврат якоря продвигается только во время активной симуляции. Ручная пауза и экран карточек замораживают возврат.

---

## 5. Повторная установка

После завершения возврата якорь переходит в `STOWED` и снова доступен для установки.

Новая успешная установка создаёт новый рабочий трос и полностью восстанавливает его прочность.

Прочность не восстанавливается:

- от отмены перегрузки;
- во время обычного удержания;
- от лечения защитников;
- от восстановления щита.

---

## 6. Публичный интерфейс домена

Внешние системы не изменяют runtime напрямую.

Они используют `AnchorSystem`:

```text
apply_rope_damage(anchor_id, amount, source)
get_rope_snapshot(anchor_id)
get_all_rope_snapshots()
get_anchor_state(anchor_id)
```

Основные события:

```text
rope_durability_changed(anchor_id, current, maximum)
rope_destroyed(anchor_id, source)
anchor_recovery_started(anchor_id, source, removed_enemy_count)
```

Подрывник троса наносит урон только через этот API и не выполняет физический обрыв самостоятельно.

---

## 7. Визуальная обратная связь

Для установленного троса показываются:

- цвет троса по состоянию прочности;
- компактная шкала;
- процент текущей прочности;
- оранжевое повреждённое состояние;
- красное мигающее критическое состояние;
- отдельное красное состояние перегрузки.

Стартовые пороги находятся в `AnchorBalance`:

```text
rope_damaged_ratio = 0.50
rope_critical_ratio = 0.25
```

Фаза мигания продвигается только при активной симуляции и замораживается паузой.

---

## 8. Сброс забега

При создании нового runtime все четыре шкалы устанавливаются на максимальное значение.

В текущем прототипе перезапуск забега перезагружает игровую сцену и создаёт новый `AnchorRuntimeStore`, поэтому состояние прочности не переносится между забегами.

---

## 9. Тестовые инварианты

- Все четыре шкалы независимы.
- Снятый трос не принимает урон.
- Значение не опускается ниже нуля.
- Событие разрушения возникает один раз.
- Путь закрывается синхронно с разрушением.
- Удаляются только враги на соответствующем тросе.
- Враги на платформе остаются живы.
- Перегруженный разрушенный трос корректно переходит в возврат.
- Пауза замораживает возврат и предупреждающую анимацию.
- Повторная установка полностью восстанавливает трос.
- Snapshot не позволяет изменять runtime.
- Перегрузка и прочность не заменяют друг друга.
