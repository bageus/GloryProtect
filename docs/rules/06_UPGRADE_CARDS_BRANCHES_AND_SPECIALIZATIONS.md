# Система карточек улучшений, веток и специализаций

Этот документ является источником правды для формирования предложений улучшений, весов направлений, условий открытия карточек, специализаций и повторяемых улучшений.

Связанные исходные наброски: issues #3–#13.

---

## 1. Основной цикл покупки

- При каждой обычной покупке игроку показываются **3 разные карточки**.
- Игрок выбирает ровно одну карточку.
- Стоимость списывается после выбора.
- Экран карточек полностью останавливает игровую симуляцию.
- Если после выбора хватает монет на следующую покупку, следующее событие начинается сразу без снятия паузы.
- В одном предложении могут находиться две или три карточки одной ветки.
- Одна и та же карточка не может занимать несколько мест одного предложения, даже если она повторяемая.
- Перед обычной генерацией система всегда проверяет готовые события специализации.

## 2. Типы карточек

### 2.1. Открывающие карточки

Открывающая карточка создаёт роль или объект и открывает базовые улучшения соответствующей ветки.

К ним относятся:

- `Пост турели`;
- `Стрелок`;
- `Пост лекаря`.

Открывающие карточки:

- не считаются одной из двух карточек для специализации;
- не закрывают линию `базовое → продвинутое`;
- не открывают индивидуальные карточки;
- не повышают вес ветки после выбора.

### 2.2. Базовые карточки

- Базовые карточки доступны с начала забега, если у ветки нет обязательной открывающей карточки.
- Для турелей, стрелков и лекаря базовые карточки заблокированы до первого выбора соответствующей открывающей карточки.
- Базовая карточка выбирается один раз, если отдельно не указана повторяемость.

### 2.3. Продвинутые карточки

- Продвинутая карточка принадлежит конкретной базовой карточке.
- Она не может попасть в пул раньше своей базовой карточки.
- Базовая и связанная продвинутая карточки образуют закрытую линию.

Пример:

```text
Усиленная атака турели → Мощная атака турели
```

### 2.4. Индивидуальные карточки ветки

- Индивидуальные карточки входят в пул после полного закрытия хотя бы одной линии `базовое → продвинутое` в своей ветке.
- После этого они доступны независимо друг от друга, если выполнены их собственные дополнительные условия.
- Они не являются специализациями и продолжают выпадать после специализации.

### 2.5. Карточки специализации

- Каждая ветка имеет три взаимоисключающие специализации.
- Выбор специализации сразу применяет её базовый эффект.
- Две невыбранные специализации блокируются до конца забега.
- После выбора специализации дополнительные карточки только выбранной специализации входят в обычный пул.
- Базовые, продвинутые и индивидуальные карточки основной ветки продолжают выпадать.

### 2.6. Дополнительные карточки специализации

Дополнительные карточки одной специализации независимы:

```text
специализация → дополнительная карточка A
специализация → дополнительная карточка B
```

Карточка A не является условием для карточки B. Количество дополнительных карточек может различаться у разных специализаций, если это не ломает баланс.

### 2.7. Общие карточки

Общие карточки:

- не принадлежат ни одной ветке;
- не изменяют веса веток;
- не считаются для специализации;
- не открывают индивидуальные карточки;
- используют отдельный общий пул с постоянным базовым весом.

Если общая карточка имеет базовую и продвинутую версии, продвинутая также требует выбора базовой.

## 3. Прогресс специализации

### 3.1. Условие готовности

Ветка становится готовой к специализации после выбора **двух засчитываемых карточек этой ветки**.

Засчитываются:

- базовые карточки;
- продвинутые карточки;
- индивидуальные карточки, если они уже были доступны.

Не засчитываются:

- открывающие карточки;
- общие карточки;
- карточки других веток.

### 3.2. Событие специализации

При следующей покупке после готовности ветки:

1. обычная генерация трёх карточек не выполняется;
2. игроку показываются все три специализации выбранной ветки;
3. игрок выбирает одну специализацию;
4. покупка оплачивается по обычной текущей стоимости;
5. счётчик стоимости переходит к следующей покупке;
6. базовый эффект специализации применяется сразу;
7. дополнительные карточки выбранной специализации входят в пул.

### 3.3. Несколько готовых веток

- Если готовы несколько веток, для текущего события одна выбирается случайно.
- Остальные сохраняют состояние готовности.
- Они участвуют в случайном выборе при следующих событиях специализации.
- Готовая ветка не может потерять право на специализацию.

Если после выбора специализации хватает монет на следующую покупку, система снова сначала проверяет готовые ветки.

### 3.4. Влияние специализации на веса

- Выбор специализации считается выбором карточки своей ветки для изменения весов.
- Дополнительные карточки специализации также изменяют веса как карточки своей ветки.
- Специализация не закрывает линию `базовое → продвинутое`.

## 4. Весовая система направлений

### 4.1. Начальные веса

В начале забега:

- каждая игровая ветка имеет вес `10`;
- общий пул имеет вес `10`;
- вес общего пула не меняется в течение забега.

Ветки:

1. турели;
2. защитники ближнего боя;
3. стрелки;
4. лекарь;
5. управление без якорей;
6. усиленные и боевые якоря;
7. щит и энергетическое ядро.

### 4.2. Изменение весов

После выбора обычной веточной карточки, специализации или карточки специализации:

- выбранная ветка получает `+3`;
- каждая связанная ветка получает `+1`;
- каждая противоположная ветка получает `−1`;
- вес ветки не может опуститься ниже `2`.

Открывающие и общие карточки веса не изменяют. Все значения должны храниться в типизированных data-driven ресурсах.

### 4.3. Связи направлений

| Выбранная ветка | Связанные ветки `+1` | Противоположные ветки `−1` |
|---|---|---|
| Турели | Якоря, лекарь | Управление без якорей |
| Ближний бой | Лекарь, якоря | — |
| Стрелки | Управление без якорей | — |
| Лекарь | Ближний бой, турели | — |
| Управление без якорей | Стрелки, щит | Якоря, турели |
| Якоря | Турели, ближний бой | Управление без якорей |
| Щит и ядро | Управление без якорей | — |

Связи являются стартовым балансом, а не жёсткими запретами.

## 5. Формирование обычного предложения

Обычное предложение строится в два этапа, чтобы ветка с большим числом карточек не получала скрытое преимущество.

### 5.1. Подготовка допустимых карточек

Для каждой карточки проверяются:

- открыта ли её ветка;
- выполнены ли предварительные карточки;
- не выбрана ли она ранее;
- не достигнут ли лимит повторений;
- выбрана ли требуемая специализация;
- не заблокирована ли альтернативная специализация;
- выполнены ли дополнительные условия карточки.

Пулы без допустимых карточек исключаются из текущего розыгрыша.

### 5.2. Заполнение трёх мест

Для каждого места:

1. по весам выбирается ветка или общий пул;
2. внутри выбранного пула случайно выбирается допустимая карточка;
3. выбранный `card_id` удаляется только из текущего предложения;
4. следующий слот разыгрывается заново.

Одна ветка может заполнить все три места. Повторяемая карточка остаётся доступной в будущих предложениях, но не дублируется внутри одного предложения.

Если осталось меньше трёх уникальных допустимых карточек, интерфейс показывает все оставшиеся и не создаёт дубликаты.

## 6. Повторяемые карточки и ограничения

### 6.1. Добавить защитника

- Начальный экипаж: `3` защитника.
- Карточка добавляет `1` защитника.
- Максимум выборов: `5`.
- Максимальный экипаж: `8` защитников.
- После пятого выбора карточка удаляется из пула.

### 6.2. Пост турели

`Пост турели` является повторяемой открывающей карточкой.

- Максимум выборов: `3`.
- Каждый выбор добавляет одну доступную турель.
- Первый выбор дополнительно открывает три базовые линии турельной ветки.
- Второй и третий выборы только добавляют ещё по одной турели.
- Ни один выбор не считается для специализации.
- Выбор карточки не изменяет вес турельной ветки.
- После третьего выбора карточка удаляется из пула.

### 6.3. Четвёртая турель

Индивидуальная карточка `4-я турель` может попасть в пул только когда одновременно:

- выбраны все три копии `Поста турели`;
- выбрана специализация турельной ветки;
- закрыта хотя бы одна турельная линия `базовое → продвинутое`;
- карточка ещё не выбрана.

Карточка выбирается один раз и увеличивает максимум доступных турелей до четырёх.

### 6.4. Стрелок

- `Стрелок` выбирается один раз.
- Карточка открывает роль стрелка и три базовые линии ветки.
- Она не считается для специализации и не изменяет вес ветки.

### 6.5. Пост лекаря

- `Пост лекаря` выбирается один раз.
- Карточка открывает медицинский пост, роль лекаря и три базовые линии ветки.
- Она не считается для специализации и не изменяет вес ветки.

| Скоростной перелёт | `+15%` к ускорению и скорости; резкое торможение при подключении | После длительного перелёта первое подключение восстанавливает `10%`; платформа сбивает врагов перед собой |
| Мощная стабилизация | При подключении разряд наносит `1` урон врагам возле последнего якоря с каждой стороны | Наземное ядро наносит `2` урона наземным врагам; ядро платформы наносит `2` урона врагам на платформе и в воздухе |

### 7.6. Усиленные и боевые якоря

**Базовые и продвинутые линии**

| Базовая карточка | Эффект | Продвинутая карточка | Эффект |
|---|---|---|---|
| Усиленный якорь | `+1` секунда к окну перегрузки | Мега-якорь | ещё `+1` секунда |
| Электрифицированные якоря | Враги на тросе получают `1` урон примерно раз в `4` секунды | Усиленные электрифицированные якоря | Усиливает периодический эффект; точное значение задаётся балансом |
| Ускоренная установка якоря | `+20%` к скорости установки | Турбо-установка якоря | ещё `+20%` |

**Индивидуальная карточка**

| Карточка | Эффект |
|---|---|
| Мгновенное снятие | Экстренная команда снимает все якоря одним действием |

**Специализации**

| Специализация | Базовый эффект | Дополнительные карточки |
|---|---|---|
| Крепкие якоря | ещё `+1` секунда к окну перегрузки | Второй якорь устанавливается на `50%` быстрее; `25%` шанс самопроизвольного падения врага |
| Электрифицированные якоря | При установке и снятии импульс наносит `1` урон и оглушает на `2` секунды с базовым шансом `30%` | Электрификация получает `50%` шанс сбросить врага |
| Якоря-ловушки | При снятии враги у края получают `1` урон и отбрасываются | При закреплении происходит взрыв с уроном `1` |

### 7.7. Щит и энергетическое ядро

**Базовые и продвинутые линии**

| Базовая карточка | Эффект | Продвинутая карточка | Эффект |
|---|---|---|---|
| Улучшенная защита щита | `+10%` к максимальной прочности | Мощная защита щита | ещё `+10%` |
| Ускоренное восстановление щита | `+10%` к скорости восстановления | Турбо-восстановление щита | ещё `+10%` |
| Расширенный контакт | `+10%` к ширине зоны контакта | Мега-контакт | ещё `+10%` |

У ветки пока нет индивидуальных карточек.

**Специализации**

| Специализация | Базовый эффект | Дополнительные карточки |
|---|---|---|
| Сосредоточенный заряд | Текущая секция восстанавливается на `15%` быстрее | `30%` врагов, атакующих текущую секцию на миникарте, меняют цель |

## 8. Общий пул

| Карточка | Тип | Эффект | Повторяемость |
|---|---|---|---|
| Добавить защитника | самостоятельная | `+1` защитник | до `5` раз |
| Улучшенная скорость перемещения защитников | базовая | `+15%` к скорости перемещения | один раз |
| Мощная скорость перемещения защитников | продвинутая | ещё `+15%` | после базовой |
| Ускоренный респаун защитников | базовая | `−25%` к времени замены | один раз |
| Турбо-респаун защитников | продвинутая | ещё `−25%` | после базовой |

Общий пул имеет постоянный вес `10` независимо от количества уже выбранных общих карточек.

## 9. Межветочные синергии

Сочетания из issue #12 пока являются примерами возможных билдов, а не отдельными карточками:

- турели + якоря;
- защитники + лекарь;
- стрелки + управление без якорей;
- щит + управление без якорей;
- турели + лекарь;
- якоря + ближний бой.

Отдельные синергетические карточки могут быть добавлены позже, если этого потребуют игровые тесты и баланс.

## 10. Требования к реализации

- Все определения карточек должны быть data-driven.
- У каждой карточки должен быть стабильный `card_id`.
- Предварительные условия задаются данными, а не цепочками `if` в UI.
- Генератор предложения не применяет эффекты карточек.
- UI отображает предложение и отправляет выбранный `card_id`.
- Выбранные карточки, повторения, веса, готовые специализации и заблокированные подветки имеют одного владельца runtime-состояния улучшений.
- Весовые значения и связи веток находятся в типизированном балансном ресурсе.
- Генератор должен быть детерминированно тестируемым с заданным seed.
- UI не хранит копии весов или прогресса специализаций.

Минимальные автоматические сценарии:

1. предложение содержит три уникальные карточки;
2. три карточки могут принадлежать одной ветке;
3. продвинутая карточка не появляется раньше базовой;
4. индивидуальная карточка не появляется до закрытия линии;
5. открывающие карточки не считаются для специализации;
6. после двух засчитываемых карточек следующая покупка становится событием специализации;
7. при нескольких готовых ветках одна выбирается случайно, остальные сохраняются;
8. после специализации альтернативные подветки не появляются;
9. дополнительные карточки специализации независимы;
10. общий пул не изменяет веса веток;
11. `Добавить защитника` исчезает после пятого выбора;
12. `Пост турели` исчезает после третьего выбора;
13. первая копия `Поста турели` открывает базовые турельные карточки;
14. `4-я турель` требует трёх постов, турельной специализации и закрытой линии;
15. повторяемая карточка не дублируется внутри одного предложения.
