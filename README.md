# GloryProtect

GloryProtect — survival roguelike с видом сбоку, в котором игрок управляет магически парящей платформой, поддерживает пять секций глобального защитного щита и защищает экипаж от физического абордажа.

## Движок

Проект разрабатывается на **Godot 4.6.2 stable** с использованием строго типизированного GDScript.

## Обязательные документы

Перед изменением кода или сцен необходимо прочитать:

- [Правила проекта](PROJECT_RULES.md)
- [Архитектура проекта](ARCHITECTURE.md)
- [Инструкции для coding agents](AGENTS.md)

Ключевые ограничения:

- запрещены монолитные классы, сцены и менеджеры;
- максимум 600 строк на поддерживаемый файл;
- с 450 строк файл должен оцениваться на разделение;
- каждое изменяемое состояние имеет одного владельца;
- настраиваемые значения находятся в ресурсах данных;
- UI не владеет игровой логикой.

## Основная идея

Платформа перемещается только по горизонтали и находится под воздействием ветра. Для восстановления секции щита игрок должен добраться до связанного наземного шара и удерживать энергетический контакт.

Якоря стабилизируют платформу, но каждый трос становится маршрутом физического абордажа. Ограниченный экипаж распределяется между управлением, якорями и боем.

## Текущая реализация

**Prototype 1.3** содержит:

- состояние забега, стартовую задержку и полную паузу;
- data-driven платформу и ветер силы 1–3;
- четыре декомпозированных якоря;
- пять фиксированных наземных шаров и секций щита;
- три физических защитника и перемещение между ролями;
- независимые кулдауны замены;
- физический абордаж только через установленные якоря;
- очереди, разделение и прыжок врагов через защитника;
- ближний бой с фиксированной целью взмаха;
- экономику забега и награды за физических врагов;
- две одинаковые карточки-заглушки без эффектов;
- общую сложность забега;
- плавный рост лимита и частоты физического абордажа;
- агрегированные стратегические волны;
- случайное распределение волны между секциями щита;
- поочерёдный урон от каждого стратегического врага;
- постоянно видимую стратегическую миникарту;
- отсутствие монет за стратегических врагов;
- регрессионные тесты.

## Сложность забега

Сложность растёт только в `RUNNING`. Стартовая задержка, ручная пауза, выбор карточек и завершённый забег не увеличивают её.

```text
Время до максимальной сложности: 600 секунд
Нормализованная сложность: 0.0 → 1.0
Наземный лимит: 8 → 20 врагов
Интервал физического спавна: 3.0 → 0.8 секунды
```

Параметры находятся в:

```text
resources/balance/run_difficulty_balance.tres
resources/balance/boarding_balance.tres
```

## Стратегические волны

Стратегические враги существуют как агрегированные группы данных и не создаются как физические `Node2D`.

Текущие диапазоны:

```text
Первая волна: через 5 секунд активной игры
Интервал волн: 12 → 4 секунды
Размер волны: 6 → 30 врагов
Время движения к щиту: 8 → 4 секунды
Число атакуемых секций: 1 → 3
Урон одного врага: 1
Интервал ударов группы: 0.35 секунды
```

Одна волна случайно выбирает уникальные секции и распределяет между ними своё реальное количество врагов. После прибытия группа остаётся у щита и уменьшается по одному врагу после каждого импульса урона.

Параметры находятся в:

```text
resources/balance/strategic_wave_balance.tres
```

Стратегические группы:

- не участвуют в ближнем бою;
- не используют якоря;
- не регистрируются в `BoardingEnemyRegistry`;
- не дают монет.

## Стратегическая миникарта

Миникарта постоянно находится в правом нижнем углу.

Она показывает:

- пять секций щита;
- текущие проценты прочности;
- зелёное, оранжевое и мигающее красное состояние;
- движение агрегированных групп;
- количество врагов внутри каждой массы;
- текущий номер волны;
- время до следующей волны;
- размер следующей волны.

## Экономика прототипа

```text
Стартовые монеты: 0
Награда за базового физического врага: 1
```

Награждаемые причины смерти:

```text
combat
anchor_path_closed
```

Стратегические враги миникарты монет не дают.

## Временные карточки

Карточки открываются автоматически при накоплении нужной суммы.

```text
5, 10, 15, …, 100, 200, 400, 800
```

Обе карточки пока одинаковы и **не применяют улучшение**. Выбор только списывает стоимость и увеличивает номер покупки.

Выбор: мышь или клавиши `1` и `2`.

## Запуск

1. Установить Godot 4.6.2 stable.
2. Клонировать репозиторий.
3. Открыть `project.godot`.
4. Запустить проект клавишей `F5`.

## Управление

- `←` / `→` — рулевое усилие.
- `1–4` — установить или снять якорь.
- `R` — снять все якоря.
- `Esc` — пауза.
- `5`, `6`, `7` — выбрать защитника.
- `D` — рулевой.
- `Z` / `X` — левый / правый якорщик.
- `C` — свободный защитник.
- `F1–F5` — выбрать секцию щита.
- `Space` — тестовый урон секции.
