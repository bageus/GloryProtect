# GloryProtect

GloryProtect — survival roguelike с видом сбоку, в котором игрок управляет магически парящей платформой, поддерживает пять секций глобального щита и защищает экипаж от физического абордажа.

## Движок

Godot **4.6.2 stable**, строго типизированный GDScript.

## Обязательные документы

- [Правила проекта](PROJECT_RULES.md)
- [Архитектура проекта](ARCHITECTURE.md)
- [Инструкции для coding agents](AGENTS.md)

Основные ограничения:

- максимум 600 строк на поддерживаемый файл;
- каждое изменяемое состояние имеет одного владельца;
- баланс находится в типизированных ресурсах;
- UI не владеет игровой логикой.

## Текущая версия

**Prototype 1.9** содержит:

- горизонтальную платформу и ветер силы 1–3;
- четыре якоря с конечной длиной троса;
- пять наземных шаров и секций щита;
- три защитника, роли, замены и ближний бой;
- физический абордаж через якоря;
- стратегические волны и миникарту;
- экономику, статистику и итоговый экран;
- сетку размещаемых объектов;
- медицинский пост и роль лекаря;
- несколько независимых турелей;
- отдельного оператора для каждой турели;
- автоматическую стрельбу и визуальные эффекты турелей;
- полноценную мышиную панель выбора экипажа и назначения ролей;
- выбор защитника кликом по персонажу или портрету;
- отображение занятых и недоступных постов;
- две одинаковые карточки-заглушки без эффектов.

## Интерфейс экипажа

Новая панель в нижней части экрана содержит:

- кнопки всех защитников;
- здоровье и текущую роль каждого;
- таймер замены погибшего защитника;
- выбранного защитника и его текущее действие;
- кнопки рулевого, левого и правого якорщика;
- кнопку свободного бойца;
- кнопку лекаря, когда медицинский пост установлен;
- отдельную кнопку каждой установленной турели;
- клетку и владельца каждого рабочего поста;
- текстовую причину отказа команды.

Выбранного защитника можно сменить:

- нажатием на его портрет;
- кликом непосредственно по персонажу на платформе;
- клавишами `5`, `6`, `7`.

Выбранный персонаж получает жёлтое кольцо. Мышь, портреты и клавиатура используют одно состояние `CrewSelectionController`.

Во время карточек панель скрывается. Во время ручной паузы она остаётся видимой, но все команды недоступны.

Старая диагностическая панель скрыта по умолчанию и переключается клавишей `F10`.

## Назначение ролей

Стандартные роли назначаются одной кнопкой:

```text
Свободный боец
Рулевой
Левый якорщик
Правый якорщик
Лекарь
```

Для каждой турели отображается отдельная кнопка вида:

```text
Турель T2 • клетка 15
занято защитником 3
```

Интерфейс блокирует кнопку, если:

- пост ещё не установлен;
- пост занят другим защитником;
- выбранный защитник погиб;
- защитник уже движется к роли;
- защитник ожидает завершения неделимого действия;
- выбранная роль уже является текущей;
- игра находится на паузе или экране карточек.

Окончательное решение всё равно принимает `CrewRoleManager`; UI только отправляет команду и показывает результат.

## Размещаемые объекты

`BuildableGrid` остаётся единственным владельцем занятости клеток и размещённых объектов.

- одна клетка содержит не больше одного объекта;
- служебные клетки недоступны;
- персонаж на клетке не мешает размещению;
- объекты не блокируют движение;
- открытие не теряется после демонтажа;
- медицинский пост существует максимум в одном экземпляре;
- в техническом прототипе можно открыть до четырёх турелей.

Координаты клеток вычисляет только `PlatformController`.

## Медицинский пост

Лекарь выбирает самого тяжело раненого живого защитника, при равенстве — ближайшего. После пяти секунд непрерывного лечения восстанавливается один сегмент. Начатый цикл завершается перед переназначением.

При переносе поста текущий цикл завершается, затем лекарь физически перебегает к новой позиции. При демонтаже он немедленно становится свободным бойцом.

## Турели

Каждая турель:

- является отдельным постом `TURRET / buildable_id`;
- требует собственного оператора;
- работает только после его прибытия;
- выбирает ближайшего врага в круговом радиусе;
- атакует врагов на тросах, платформе и в прыжке;
- наносит 1 урон;
- не перенаправляет начатый выстрел;
- имеет собственные подготовку и кулдаун.

Выбранная турель показывает радиус. Визуал включает заряд, отдачу, вспышку и короткий трассер.

## Карточки

Стоимость:

```text
5, 10, 15, …, 100, 200, 400, 800
```

Обе карточки пока одинаковы и не применяют улучшение. Медицинский пост и турели открываются debug-командами.

## Управление прототипом

Основное управление экипажем теперь выполняется мышью.

Дополнительные клавиши:

- `←` / `→` — рулевое усилие;
- `1–4` — установить или снять якорь;
- `R` — снять все якоря;
- `Esc` — пауза;
- `5`, `6`, `7` — выбрать защитника;
- `D` — рулевой;
- `Z` / `X` — левый / правый якорщик;
- `C` — свободный защитник;
- `H` — лекарь;
- `,` / `.` — выбрать клетку;
- `B`, `M`, `Delete` — медицинский пост;
- `T`, `G`, `K`, `V`, `J`, `Backspace` — debug-команды турелей;
- `F10` — диагностическая панель;
- `Enter` — новый забег после поражения.
