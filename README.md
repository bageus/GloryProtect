# GloryProtect

GloryProtect — survival roguelike с видом сбоку, в котором игрок управляет магически парящей платформой, поддерживает пять секций глобального защитного щита и защищает экипаж от непрерывного абордажа.

## Движок

Проект разрабатывается на **Godot 4.6.2 stable** с использованием GDScript.

## Обязательные документы разработки

Перед изменением кода или сцен необходимо прочитать:

- [Правила проекта](PROJECT_RULES.md)
- [Архитектура проекта](ARCHITECTURE.md)
- [Инструкции для coding agents](AGENTS.md)

Ключевые ограничения:

- запрещены монолитные классы, сцены и менеджеры;
- ни один исходный файл не может превышать 600 строк;
- примерно с 450 строк файл должен оцениваться на разделение;
- каждое изменяемое состояние имеет одного владельца;
- настраиваемые значения выносятся в ресурсы данных;
- UI не владеет игровой логикой;
- архитектурные изменения обновляют `ARCHITECTURE.md` в том же коммите.

## Основная идея

Платформа перемещается только по горизонтали и находится под воздействием ветра. Для восстановления секции глобального щита игрок должен добраться до связанного наземного шара и удерживать энергетический контакт.

Якоря стабилизируют платформу, но каждый установленный трос становится маршрутом физического абордажа. Ограниченный экипаж распределяется между управлением, якорями, боем, лечением и турелями.

## Текущая реализация

**Prototype 0.4** содержит:

- состояние забега и полную паузу;
- data-driven ветер силы 1–3;
- физическое рулевое усилие, инерцию и границы;
- отдельные `PlatformController`, `SteeringInputProvider` и визуальный компонент;
- тестовый энергетический контакт;
- четыре декомпозированных якоря;
- очередь установки и экстренное снятие;
- конечную длину троса в обоих направлениях;
- перегрузку одиночного натянутого якоря при ветре силы 3;
- три физических защитника на платформе;
- три сегмента здоровья у каждого защитника;
- отдельные `CrewManager`, `CrewRoleManager` и `RoleStationRegistry`;
- физическое перемещение между рабочими местами;
- отключение старого поста во время перехода;
- включение новой роли только после прибытия;
- отложенный уход рулевого до отпускания ввода;
- завершение текущей установки перед уходом якорщика;
- unit- и integration-сценарии для якорей, ресурсов и ролей.

## Запуск

1. Установить Godot 4.6.2 stable.
2. Клонировать репозиторий.
3. Открыть `project.godot`.
4. Запустить проект клавишей `F5`.

## Управление платформой

- `←` / `→` — рулевое усилие.
- `1–4` — установить или снять соответствующий якорь.
- `R` — снять все якоря.
- `Space` — нанести тестовой секции 10 урона.
- `Esc` — пауза.

## Временное управление экипажем

- `5`, `6`, `7` — выбрать защитника.
- `D` — назначить рулевым.
- `Z` — назначить левым якорщиком.
- `X` — назначить правым якорщиком.
- `C` — назначить свободным защитником.

Финальный интерфейс будет использовать выбор персонажа или его карточки и контекстное меню ролей.
