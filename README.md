# GloryProtect

GloryProtect — survival roguelike с видом сбоку, в котором игрок управляет магически парящей платформой, поддерживает пять секций глобального щита и защищает экипаж от физического абордажа.

## Движок

Godot **4.6.2 stable**, строго типизированный GDScript.

## Обязательные документы

- [Правила проекта](PROJECT_RULES.md)
- [Архитектура проекта](ARCHITECTURE.md)
- [Инструкции для coding agents](AGENTS.md)

Основные ограничения:

- максимум 600 строк на поддерживаемый файл;
- каждое изменяемое состояние имеет одного владельца;
- баланс находится в типизированных ресурсах;
- UI не владеет игровой логикой.

## Текущая версия

**Prototype 2.1 foundation** содержит:

- горизонтальную платформу и ветер силы 1–3;
- четыре якоря с конечной длиной троса;
- независимую runtime-прочность каждого из четырёх тросов;
- публичный API урона тросу, snapshots и события разрушения;
- пять наземных шаров и секций щита;
- три защитника, роли, замены и ближний бой;
- физический абордаж через якоря;
- три data-driven типа физических врагов;
- состав абордажа, меняющийся с ростом сложности;
- стратегические волны и миникарту;
- экономику, статистику и итоговый экран;
- медицинский пост и несколько независимых турелей;
- мышиную панель выбора экипажа и назначения ролей;
- две одинаковые карточки-заглушки без эффектов.

Нулевая прочность троса пока только публикует доменное событие. Физический обрыв, закрытие пути, падение врагов и возврат якоря относятся к следующей отдельной задаче.

## Прочность тросов

Максимальная прочность задаётся в `AnchorBalance`. Текущее значение хранится отдельно у каждого `AnchorRuntime`, а изменяет его только `AnchorRopeDurability`.

Внешние системы используют `AnchorSystem`:

```text
apply_rope_damage(anchor_id, amount, source)
get_rope_snapshot(anchor_id)
get_all_rope_snapshots()
```

Новая успешная установка полностью восстанавливает прочность соответствующего троса. Повреждение прочности не заменяет и не изменяет механику перегрузки ветром.

## Типы физических врагов

Все текущие типы используют одну проверенную машину состояний:

```text
земля → выбранный якорь → подъём → платформа → ближний бой
```

Различия задаются только ресурсами `BoardingEnemyArchetype`.

### Базовый абордажник

```text
Здоровье: 1
Скорость по земле: 125
Скорость подъёма: 105
Скорость на платформе: 90
Радиус тела: 12
Появляется: с начала забега
```

### Быстрый абордажник

```text
Здоровье: 1
Скорость по земле: 175
Скорость подъёма: 150
Скорость на платформе: 135
Радиус тела: 10
Более короткая подготовка атаки
Появляется: с 15% общей сложности
```

### Тяжёлый абордажник

```text
Здоровье: 3
Скорость по земле: 85
Скорость подъёма: 70
Скорость на платформе: 62
Радиус тела: 16
Более медленная атака
Появляется: с 45% общей сложности
```

В начале автоматический спавн может выбрать только базовый тип. По мере сложности вес быстрого и тяжёлого типов растёт. Выбор выполняет `BoardingEnemyCatalog`, а не условные ветки в `BoardingEnemyController`.

Пока все три типа дают одинаковую базовую награду за физическое убийство. Стратегические враги миникарты монет не дают.

## Данные врагов

```text
scripts/boarding/boarding_enemy_archetype.gd
scripts/boarding/boarding_enemy_catalog.gd
resources/enemies/boarding_basic.tres
resources/enemies/boarding_runner.tres
resources/enemies/boarding_brute.tres
resources/enemies/boarding_enemy_catalog.tres
```

Архетип задаёт:

- стабильный `archetype_id`;
- отображаемое имя;
- здоровье;
- радиус тела;
- скорости на земле, тросе и платформе;
- урон, подготовку, кулдаун и дальность атаки;
- диагностические цвета;
- порог открытия и вес спавна.

`BoardingBalance` продолжает владеть общими правилами абордажа: интервалом спавна, лимитом наземных врагов, минимальными интервалами, прыжком и параметрами защитников.

## Физическое разделение

Расстояние между двумя врагами теперь вычисляется как максимум из:

```text
глобальный минимальный интервал
сумма радиусов двух конкретных архетипов
```

Это правило применяется:

- на земле;
- в очереди на тросе;
- при выходе на платформу;
- при движении на платформе;
- при резервировании точки приземления прыжка.

Тяжёлые враги поэтому занимают больше места и создают более плотные пробки.

## Диагностический внешний вид

Финальные ассеты ещё не подключены. Временная векторная графика различает типы:

- базовый — красный;
- быстрый — фиолетовый и более компактный;
- тяжёлый — коричнево-оранжевый и крупный.

Это только представление. Цвет и маркер не участвуют в логике.

## Интерфейс экипажа

Защитника можно выбрать кликом по персонажу, портретом или клавишами `5`, `6`, `7`. Панель показывает здоровье, текущую и целевую роль, занятость постов, медицинский пост и каждую конкретную турель.

Во время карточек панель скрывается. Во время ручной паузы она остаётся видимой, но команды отключены. Диагностическая телеметрия переключается `F10`.

## Карточки

Стоимость остаётся:

```text
5, 10, 15, …, 100, 200, 400, 800
```

Обе карточки пока одинаковы и не применяют улучшение. Переход на новую систему трёх карточек реализуется отдельными issues upgrade foundation.

## Управление прототипом

Основное управление экипажем выполняется мышью.

Дополнительные клавиши:

- `←` / `→` — рулевое усилие;
- `1–4` — установить или снять якорь;
- `R` — снять все якоря;
- `Esc` — пауза;
- `5`, `6`, `7` — выбрать защитника;
- `D`, `Z`, `X`, `C`, `H` — debug-назначения ролей;
- `,` / `.` — выбрать клетку;
- `B`, `M`, `Delete` — медицинский пост;
- `T`, `G`, `K`, `V`, `J`, `Backspace` — debug-команды турелей;
- `F10` — диагностическая панель;
- `Enter` — новый забег после поражения.
