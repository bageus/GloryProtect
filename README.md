# GloryProtect

GloryProtect — survival roguelike с видом сбоку, в котором игрок управляет магически парящей платформой, поддерживает пять секций глобального щита и защищает экипаж от физического абордажа.

## Движок

Godot **4.6.2 stable**, строго типизированный GDScript.

Основные ограничения:

- максимум 600 строк на поддерживаемый файл;
- каждое изменяемое состояние имеет одного владельца;
- баланс находится в типизированных ресурсах;
- UI не владеет игровой логикой.

## Текущая версия

**Prototype 2.1 foundation** содержит:

- горизонтальную платформу и ветер силы 1–3;
- четыре якоря с независимой прочностью тросов;
- визуальную шкалу, критическое предупреждение и физический обрыв троса;
- автоматический возврат разрушенного якоря;
- подрывника, атакующего установленный трос с земли;
- пять наземных шаров и секций щита;
- три стартовых защитника с возможностью увеличить экипаж до восьми;
- роли, замены, ближний и дальний бой;
- физический абордаж через якоря;
- data-driven каталог физических врагов;
- подключаемые специальные поведения врагов;
- стратегические волны и миникарту;
- экономику, статистику и итоговый экран;
- медицинский пост и несколько независимых турелей;
- систему трёх карточек, весов веток и событий специализации;
- общий пул и повторяемые карточки.

## Прочность тросов

Максимальная прочность задаётся в `AnchorBalance`. Текущее значение хранится отдельно у каждого `AnchorRuntime`, а изменяет его только `AnchorRopeDurability`.

Внешние системы используют `AnchorSystem`:

```text
apply_rope_damage(anchor_id, amount, source)
get_rope_snapshot(anchor_id)
get_all_rope_snapshots()
get_anchor_state(anchor_id)
```

`AnchorBreakRecoveryController` отвечает за физические последствия нуля: закрывает путь, удаляет врагов с конкретного троса через общий death flow, переводит якорь в `RETURNING` и пересчитывает ограничения платформы.

## Типы физических врагов

Обычные абордажники используют общую машину состояний:

```text
земля → выбранный якорь → подъём → платформа → ближний бой
```

Специальные враги подключают отдельный `EnemyBehaviorComponent`. Их состояние, доступность для турелей и принадлежность к наземной, тросовой или бортовой группе определяет behavior, а не проверка конкретного ID.

### Базовый абордажник

```text
Здоровье: 1
Скорость по земле: 125
Скорость подъёма: 105
Скорость на платформе: 90
Радиус тела: 12
Появляется: с начала забега
```

### Быстрый абордажник

```text
Здоровье: 1
Скорость по земле: 175
Скорость подъёма: 150
Скорость на платформе: 135
Радиус тела: 10
Появляется: с 15% общей сложности
```

### Подрывник троса

```text
Здоровье: 1
Скорость по земле: 155
Урон тросу: 35
Подготовка: 1.6 секунды
Появляется: с 25% общей сложности
```

Подрывник фиксирует ближайший работоспособный трос, бежит к наземной точке крепления и после подготовки наносит урон только через публичный API. Самоподрыв не даёт награды; убийство до взрыва использует обычную награду.

### Тяжёлый абордажник

```text
Здоровье: 3
Скорость по земле: 85
Скорость подъёма: 70
Скорость на платформе: 62
Радиус тела: 16
Появляется: с 45% общей сложности
```

## Система улучшений

Каждое обычное предложение содержит до трёх разных карточек. Повторяемая карточка может появиться снова в будущем, но не занимает два места одного предложения.

Текущий игровой каталог находится в:

```text
resources/upgrades/game_upgrade_catalog.tres
```

### Общий пул

Общие карточки имеют постоянный вес и не изменяют прогресс веток:

- `Добавить защитника` — до пяти выборов, максимум восемь защитников;
- две карточки скорости перемещения — по `+15%`;
- две карточки времени замены — по `−25%`.

Множители применяются через публичные API `CrewManager` и `CrewReplacementController`. UI и каталог не изменяют персонажей напрямую.

### Повторяемый пост турели

`Пост турели` выбирается максимум три раза. Каждая копия открывает ещё одну доступную турель, а первая копия также открывает три базовые линии:

- урон;
- кулдаун;
- радиус.

Открывающая карточка не считается для специализации и не меняет вес турельной ветки. Базовый выстрел турели остаётся равным `1`.

`4-я турель` появляется только после трёх постов, выбора турельной специализации и полного закрытия хотя бы одной линии `базовое → продвинутое`.

Боевые эффекты турельных карточек и специализаций подключаются к `TurretSystem` в NEXT-09. В текущем foundation их значения уже сохраняются в `UpgradeRuntime` как data-driven domain flags/scalars.

## Физическое разделение

Расстояние между персонажами вычисляется как максимум из глобального минимального интервала и суммы радиусов конкретных сущностей. Правило применяется на земле, тросе, платформе и при резервировании точки приземления.

## Интерфейс экипажа

Защитника можно выбрать кликом по персонажу или портретом. Новые защитники автоматически регистрируются в `CrewRoleManager` как свободные бойцы и доступны в той же панели команд.

Во время карточек мировая симуляция остановлена. Во время ручной паузы панель остаётся видимой, но команды отключены. Диагностическая телеметрия переключается `F10`.

## Стоимость карточек

```text
5, 10, 15, …, 100, 200, 400, 800
```

## Управление прототипом

- `←` / `→` — рулевое усилие;
- `1–4` — установить или снять якорь;
- `R` — снять все якоря;
- `Esc` — пауза;
- `5`, `6`, `7` — выбрать одного из стартовых защитников клавиатурой;
- клик по персонажу или портрету — выбрать любого защитника;
- `D`, `Z`, `X`, `C`, `H` — debug-назначения ролей;
- `,` / `.` — выбрать клетку;
- `B`, `M`, `Delete` — медицинский пост;
- `T`, `G`, `K`, `V`, `J`, `Backspace` — debug-команды турелей;
- `F10` — диагностическая панель;
- `Enter` — новый забег после поражения.
