# GloryProtect

GloryProtect — survival roguelike с видом сбоку, в котором игрок управляет магически парящей платформой, поддерживает пять секций глобального щита и защищает экипаж от физического абордажа.

## Движок

Godot **4.6.2 stable**, строго типизированный GDScript.

## Обязательные документы

- [Правила проекта](PROJECT_RULES.md)
- [Архитектура проекта](ARCHITECTURE.md)
- [Инструкции для coding agents](AGENTS.md)

Основные ограничения:

- максимум 600 строк на поддерживаемый файл;
- каждое изменяемое состояние имеет одного владельца;
- баланс находится в типизированных ресурсах;
- UI не владеет игровой логикой.

## Текущая версия

**Prototype 1.6** содержит:

- горизонтальную платформу и ветер силы 1–3;
- четыре якоря с конечной длиной троса;
- пять наземных шаров и секций щита;
- три защитника, роли, замены и ближний бой;
- физический абордаж только через якоря;
- агрегированные стратегические волны и миникарту;
- экономику, статистику и итоговый экран;
- критическое визуальное и звуковое предупреждение;
- сетку размещаемых объектов по клеткам платформы;
- один открываемый медицинский пост;
- перенос и демонтаж медицинского поста;
- назначение обычного защитника лекарем;
- лечение самого тяжело раненого союзника;
- неделимый пятисекундный цикл лечения;
- основу типа `TURRET` без реализации турели;
- две одинаковые карточки-заглушки без эффектов.

## Размещаемые объекты

Компоненты:

```text
BuildableInventory
BuildableGrid
BuildableRuntime
BuildableSnapshot
BuildableGridVisual
```

Правила:

- одна клетка содержит не больше одного объекта;
- служебные клетки рулевого, якорщиков и двери замены недоступны;
- персонаж на клетке не мешает размещению;
- объекты не блокируют движение персонажей;
- полученное открытие не теряется после демонтажа;
- медицинский пост существует максимум в одном экземпляре.

Координаты клеток вычисляет `PlatformController`, поэтому сетка, роли и визуал не повторяют формулу положения.

## Медицинский пост

В прототипе пост открывается debug-командой и не связан с карточками.

После установки:

1. роль `MEDIC` становится доступной;
2. выбранный защитник физически идёт к посту;
3. лекарь выбирает живого защитника с минимальным здоровьем;
4. при равном здоровье выбирается ближайший;
5. лекарь добегает до цели;
6. после 5 секунд непрерывного лечения восстанавливается 1 сегмент;
7. после цикла приоритет оценивается заново;
8. если раненых нет, лекарь возвращается к посту.

Лекарь не использует ближний бой, пока роль активна. Новое назначение не прерывает уже начатый цикл: сначала восстанавливается сегмент, затем защитник уходит. Если цель выходит из радиуса, прогресс непрерывного цикла сбрасывается.

При переносе поста текущий цикл лечения продолжается. При демонтаже лечение прекращается, а назначенный лекарь немедленно становится свободным бойцом.

## Баланс медицинского поста

```text
Количество постов: 1
Интервал лечения: 5.0 секунд
Восстановление: 1 сегмент
Радиус непрерывного лечения: 18
```

Параметры находятся в:

```text
resources/balance/buildable_balance.tres
```

## Карточки

Стоимость:

```text
5, 10, 15, …, 100, 200, 400, 800
```

Обе карточки пока одинаковы и не применяют улучшение. Выбор только списывает стоимость.

## Управление прототипом

- `←` / `→` — рулевое усилие.
- `1–4` — установить или снять якорь.
- `R` — снять все якоря.
- `Esc` — пауза.
- `5`, `6`, `7` — выбрать защитника.
- `D` — рулевой.
- `Z` / `X` — левый / правый якорщик.
- `C` — свободный защитник.
- `H` — назначить выбранного защитника лекарем.
- `,` / `.` — выбрать клетку платформы.
- `B` — открыть медицинский пост.
- `M` — установить или перенести медицинский пост.
- `Delete` — демонтировать медицинский пост.
- `F1–F5` — выбрать секцию щита.
- `Space` — тестовый урон секции.
- `Enter` — новый забег на итоговом экране.
