# GloryProtect

GloryProtect — survival roguelike с видом сбоку, в котором игрок управляет магически парящей платформой, поддерживает пять секций глобального щита и защищает экипаж от физического абордажа.

## Движок

Godot **4.6.2 stable**, строго типизированный GDScript.

## Обязательные документы

- [Правила проекта](PROJECT_RULES.md)
- [Архитектура проекта](ARCHITECTURE.md)
- [Инструкции для coding agents](AGENTS.md)

Основные ограничения:

- максимум 600 строк на поддерживаемый файл;
- каждое изменяемое состояние имеет одного владельца;
- баланс находится в типизированных ресурсах;
- UI не владеет игровой логикой.

## Текущая версия

**Prototype 1.5** содержит:

- горизонтальную платформу и ветер силы 1–3;
- четыре якоря с конечной длиной троса;
- пять наземных шаров и секций щита;
- три защитника, роли, замены и ближний бой;
- физический абордаж только через якоря;
- монеты только за физических врагов;
- две одинаковые карточки-заглушки без эффектов;
- общий параметр сложности;
- агрегированные стратегические волны;
- объединение, разделение и перенаправление масс;
- постоянно видимую миникарту;
- статистику времени и физических убийств;
- итоговый экран поражения;
- лучшие результаты текущего запуска приложения;
- общий визуальный и звуковой сигнал критического щита.

## Статистика забега

`RunStatistics` учитывает:

```text
время выживания
число награждаемых физических смертей
остаток монет
число купленных карточек
причину завершения
```

Время не дублируется и читается из `RunDifficulty`. Физическое убийство учитывается по тому же событию, которое выдаёт монету. Стратегические враги в статистику убийств не входят.

HUD постоянно показывает:

```text
MM:SS | физические убийства N
```

После поражения отображаются:

- причина;
- время;
- физические убийства;
- оставшиеся монеты;
- купленные карточки;
- число завершённых забегов текущего запуска;
- лучшее время;
- максимум убийств.

Кнопка **«Новый забег»** или `Enter` полностью перезагружает игровую сцену.

## Критическое предупреждение

Когда секция впервые входит в диапазон 25% и ниже:

- появляется красное сообщение;
- воспроизводится короткий процедурный звуковой сигнал;
- несколько секций одного кадра объединяются в одно предупреждение;
- повторного звука нет, пока секция не выйдет из критического диапазона и не войдёт снова.

## Сложность и угрозы

```text
Время до максимальной сложности: 600 секунд
Наземный лимит физических врагов: 8 → 20
Интервал физического спавна: 3.0 → 0.8 секунды
Интервал стратегических волн: 12 → 4 секунды
Размер стратегической волны: 6 → 30 врагов
Атакуемые секции: 1 → 3
```

Стратегические группы существуют как агрегированные данные, не участвуют в физическом бою и не дают монет.

## Карточки

Стоимость:

```text
5, 10, 15, …, 100, 200, 400, 800
```

Обе карточки пока одинаковы и не применяют улучшение. Выбор только списывает стоимость.

## Запуск

1. Установить Godot 4.6.2 stable.
2. Клонировать репозиторий.
3. Открыть `project.godot`.
4. Запустить проект клавишей `F5`.

## Управление

- `←` / `→` — рулевое усилие.
- `1–4` — установить или снять якорь.
- `R` — снять все якоря.
- `Esc` — пауза.
- `5`, `6`, `7` — выбрать защитника.
- `D` — рулевой.
- `Z` / `X` — левый / правый якорщик.
- `C` — свободный защитник.
- `F1–F5` — выбрать секцию щита.
- `Space` — тестовый урон секции.
- `Enter` — новый забег на итоговом экране.
